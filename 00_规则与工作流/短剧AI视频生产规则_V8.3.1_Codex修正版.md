# 短剧 AI 视频生产规则 V8.3.1 Codex 修正版

规则状态：V8.3.1 当前活动执行规则源，Codex 可读取 Markdown 版。

来源：基于《短剧AI视频生产规则_V8.2_竖屏专项补丁版.md》和《短剧AI视频生产规则_V8.3_Codex.md》修正升级。V8.3.1 保留 V8.2 的九阶段流程、所见即所得、资产编号、场景锚定、覆盖镜头池、证据式二次检查、导演连续性实检、表演调度优先、真实现场拍摄光影融合原则；吸收 V8.3 的高频生产入口、首 3 秒 / 前 6 秒剪辑策略边界、参考素材用途说明、音频与字幕生产分层、竖屏平台安全区、AI 真人镜头风险分级、台词密度规则、覆盖镜头剪辑功能标签、常见失败修法映射；同时修正 V8.3 中关于流程跳转、时长分层、参考优先级标签、场景唯一参考和覆盖镜头检查粒度的执行风险。

适用范围：AI 真人竖屏短剧的剧本分析、项目生产规划、风格锁定、资产拆解、资产生图 Prompt、分镜表、Seedance2.0 分镜 Prompt、覆盖镜头池、剪辑策略检查、发布前 QC、终检与返工。

## V8.3.1 本次修正口径

1. 首 3 秒 / 前 6 秒规则只用于剧本分析、分镜排序建议、覆盖镜头池规划、剪辑成片和发布前 QC，不进入正式 Seedance2.0 镜头 Prompt。
2. 正式 Seedance2.0 镜头 Prompt 仍按剧本本身内容、分镜表编号、场景资产、人物状态、动作节点、台词和声音目标拆分。
3. 本规则制作的是剧集中的可生成镜头，不把后期剪辑时长写入正式 Seedance2.0 镜头 Prompt。每组主镜头必须给出总时长，每组主镜头总时长不超过 15 秒；覆盖镜头不计入每组主镜头总时长，但每条覆盖镜头必须有自己的持续时间。
4. Seedance2.0 单文本框结构增加“参考素材用途”，用于告诉 Seedance2.0 每个参考素材具体参考什么内容；不使用 P0 / P1 / P2，也不写“主控参考 / 辅助参考 / 局部参考”标签。
5. “不要字幕、不要音乐”只限制 Seedance2.0 生成阶段；后期字幕、配乐、转场音效和爆点音效属于剪辑与后期 QC，不进入正式 Seedance2.0 镜头 Prompt。
6. 竖屏构图规则增加平台 UI 安全区检查，防止脸、嘴型、关键道具、身份符号和字幕被遮挡。
7. 新增 AI 真人镜头风险分级 R1-R4、台词密度规则、覆盖镜头剪辑用途标签和常见失败修法映射，服务小团队高频返工。
8. Seedance2.0 按“每组独立可复制单文本框”执行，一次只生成一组 15 秒以内主镜头；每组不能依赖上一组记忆，必须在本组内重新建立资产、光影、站位、道具和声音约束。
9. 每组单文本框结构固定为：参考素材用途与资产锚定 → 全局风格与光影基准 → 本组光影继承与人物场景融合 → 镜头组正文内容 → 音频同步模式 → 生成约束 / Negative Prompt。
10. 资产锚定统一使用“镜头使用名=@资产登记名”；镜头正文只能使用等号前的镜头使用名，不能混用资产文件名、未锚定别名或描述名。
11. 每组只放本组会出现、会发声，或承担本组场景光影基准的资产；素材冲突和禁止继承只用于确有误继承风险的参考素材。
12. 镜头组之间需要连续性锚定：下一组第一镜必须锚定上一组结束镜的人物位置、动作关系和道具状态，但必须重新设计景别，不能机械延续上一组景别。


## 0. V8.3.1 总原则

### 0.1 执行模式与固定生产流程

V8.3.1 保留九阶段流程。实际从哪个阶段开始执行，由用户当轮指令、已有资料完整度和项目当前状态决定。

从中间阶段开始，不等于废弃前置阶段；必须先读取或确认已有前置成果。如果缺少会影响当前阶段判断的前置资料，必须在检查层标注缺口，并按用户指令决定是补齐、临时假设还是继续执行。

V8.3.1 支持四种执行入口模式。入口模式只决定本轮从哪里开始，不改变九阶段总框架。

模式 A：完整建制模式。适用于新项目、新主角、新世界观、第一集、风格未锁定。执行完整九阶段。

模式 B：单集快产入口。适用于项目风格、主角资产、核心场景已锁定，只制作新一集。可从单集剧本分析或分镜表阶段开始，但必须确认已有项目生产规划、风格锁定、资产清单和资产图可用。执行：

1. 单集剧本分析
2. 本集资产缺口
3. 分镜表
4. Seedance2.0 分镜 Prompt
5. 覆盖镜头池
6. 终检与返工

模式 C：镜头返工入口。适用于已有生成结果不合格。可从失败现象分析和局部修正版 Prompt 开始，但必须保持已确认的人物、场景、构图、动作、台词和资产不变。执行：

1. 失败现象分析
2. 错误类型定位
3. 局部修正版 Prompt
4. 二次生成后质检

模式 D：覆盖镜头补拍入口。适用于主线镜头已完成，但剪辑缺反应、空镜、道具、口型遮盖、身份符号或转场素材。可从覆盖镜头池开始，但必须读取主镜头尾帧、人物视线、动作方向、道具状态和场景资产。执行：

1. 缺口判断
2. 覆盖镜头池
3. Seedance2.0 覆盖镜头 Prompt
4. 剪辑接入检查

模式 A 的完整生产流程为：

1. 短剧剧本分析
2. 项目生产规划
3. 风格锁定
4. 资产拆解
5. 资产生图 Prompt
6. 分镜表
7. Seedance2.0 分镜 Prompt
8. 覆盖镜头池
9. 终检与返工

不得从剧本直接跳到最终 Seedance2.0 Prompt。每个阶段都要产出可检查的中间结果，并在关键阶段完成后等待用户确认；如果用户明确要求连续推进，可以继续执行，但必须标注已跳过确认的风险。

如果用户要求跳过阶段，应先提醒风险；用户仍要求跳过时可以执行，并标注跳过造成的风险。

### 0.2 纯 Prompt 所见即所得最高优先级

Seedance2.0 中文纯 Prompt 只写当前镜头里观众能够直接看见、直接听见的内容。

纯 Prompt 禁止写：

- 剧情解释
- 人物心理判断
- 导演意图
- 资产用途说明
- 画面外动作路径
- 规则提醒
- 二选一或不确定表达
- 模型理解辅助语
- 当前景别中不可见的表情、眼神、手指微动作
- “为了表现、用于表现、暗示、说明、代表、意味着、基于参考图、不要、避免、可以、或者、必要时”等污染词

策略层、检查层可以做导演判断、连续性判断、表演合理性判断和空间调度判断，但这些判断不能进入中文纯 Prompt。

### 0.3 创作身份只进入前置分析和检查层

可使用导演、摄影指导、灯光指导、美术指导、动作指导、声音指导、Prompt 工程架构师、AI 分镜统筹等身份做前置判断。

创作身份不得写入最终资产 Prompt 或 Seedance2.0 中文纯 Prompt。

### 0.4 全局生成约束、字幕与音乐生产分层

字幕、画面文字、背景音乐、配乐、正面美容补光、棚拍柔光箱、均匀亮脸、HDR 式提亮等禁止项，写入全局生成约束 / Negative Prompt，不写入每个中文镜头正文。

Seedance2.0 单文本框里允许有全局区。全局区不是中文纯 Prompt。

“不要字幕”只限制 Seedance2.0 生成阶段不生成字幕、标题、弹幕、屏幕文字和水印；后期字幕阶段可以添加人工字幕、角色名、集数、剧情提示和平台标题，但必须通过竖屏安全区检查。

“不要音乐”只限制 Seedance2.0 生成阶段不自动生成背景音乐和配乐；后期声音阶段可以加入配乐、转场音效、爆点音效和片头片尾声音标识，但必须单独检查版权、平台可用性、对白清晰度和音乐压对白问题。

Seedance2.0 生成阶段默认允许自然环境声、动作声、角色对白、口型同步和必要 OS。音乐、强节奏鼓点、转场音效和发布用卡点音乐默认不交给 Seedance2.0 自动生成。

### 0.5 真实现场拍摄光影融合原则

V8.3 的光影目标不是把人物脸单独提亮，而是让人物、服装、道具、地面、背景共享同一套真实现场光源关系。

光影继承优先检查人物与场景是否融合：

- 人物脸部受光方向必须与场景主光方向一致。
- 人物、服装、道具、地面和背景必须共享同一套现场光源。
- 受光面、背光面、阴影落点、色温关系和暗部层次必须符合场景资产建立的光源逻辑。
- 人物脸部允许半边落入暗部，但眼神、嘴型、面部轮廓必须在真实曝光关系内可辨，暗部不能吞脸。
- “可辨”不等于自动补光、独立柔光、均匀亮脸或 HDR 提亮暗部。
- 人物边缘光只允许来自场景内已有光源或画面内明确可见光源。

生成约束负责承载禁止项；本组光影继承负责说明人物与场景的融合关系；镜头正文只写当前镜头可见的光影结果。

### 0.6 实测失败反馈

Seedance2.0 实测后如出现明显失败，必须在终检与返工阶段记录失败类型、问题镜头、原 Prompt、失败现象、原因判断、修正版，并判断是否沉淀为规则补丁。

高频失败必须沉淀规则补丁。单次偶发失败可以只记录，不立即改规则。

### 0.7 竖屏专项启用原则

当项目画幅为 9:16，或用户明确指定竖屏短剧时，启用 V8.3.1 竖屏专项规则。

竖屏专项规则不改变 V8.3.1 九阶段主流程，只改变各阶段的判断重点和质检标准。

竖屏短剧优先保证：

- 人物脸部在手机屏幕上可读
- 眼神和嘴型可辨
- 情绪表达直接
- 构图集中，不让背景抢主体
- 场景服务人物关系和剧情信息
- 道具在中下区域可读，并服务剧情
- 镜头节奏更快，信息传达不拖
- 身份反转、情绪爆点、核心冲突在竖屏中容易读懂

竖屏专项不是把人物脸单独提亮，也不是破坏真实现场光源。人物可读性必须服从 V8.3 的真实现场拍摄光影融合原则。

竖屏专项禁止：

- 宽屏横向调度直接硬套到竖屏
- 人物太小，脸部看不清
- 主体偏到画面边缘
- 重要人物被裁掉头
- 道具位于安全区外或太小太暗
- 背景装饰抢人物
- 为了电影感牺牲手机屏幕可读性

### 0.8 首 3 秒 / 前 6 秒规则边界

首 3 秒 / 前 6 秒规则是短剧剧本分析、分镜排序建议、覆盖镜头池规划、剪辑成片和发布前 QC 的判断规则，不是 Seedance2.0 镜头 Prompt 写作规则。

可使用首 3 秒 / 前 6 秒规则判断：

- 剧本开场是否能快速建立冲突、身份、危机、误会、证据或情绪张力。
- 哪些镜头适合作为开场候选、冷开场、闪回插入或前置剪辑素材。
- 是否需要补反应镜、道具镜、身份符号镜、压迫关系镜或静音可读镜头。
- 成片首帧、首 3 秒、前 6 秒是否具有可读性。

不得使用首 3 秒 / 前 6 秒规则：

- 改写剧本原有剧情内容。
- 强行压缩正式分镜镜头。
- 在 Seedance2.0 中文镜头正文中写“钩子、爽点、爆点、留存、反转感、观众注意力”等说明性词。
- 为了开场效果破坏人物站位、动作连续性、台词顺序、道具归属、场景逻辑和尾帧状态。

如需增强开场吸引力，只能在剪辑阶段调整镜头排序，或在覆盖镜头池中补充可见、可生成、符合剧本的反应镜、道具镜、身份符号镜和压迫关系镜。

### 0.9 镜头组时长规则

V8.3.1 制作的是剧集中的生成镜头，不在正式 Seedance2.0 镜头 Prompt 中写后期剪辑使用时长。

正式 Seedance2.0 分镜 Prompt 必须遵守：

1. 每组主镜头必须给出主镜头组总时长。
2. 每组主镜头总时长不超过 15 秒；超过 15 秒必须拆组。
3. 每个主镜头必须写自己的持续时间，精确到 1 秒或 0.5 秒。
4. 覆盖镜头不计入每组主镜头总时长。
5. 每条覆盖镜头仍然是要生成的视频镜头，必须有自己的持续时间。
6. 后期剪辑如何截取、压缩、排序属于剪辑阶段，不反向改变已经通过的正式 Seedance2.0 镜头 Prompt。

推荐记录格式：

```text
主镜头组总时长：X秒
镜头一，持续X秒。
镜头二，持续X秒。
覆盖镜头池：
覆盖镜头一，持续X秒。
```

剪辑阶段可以另行判断可用区间、剪辑使用时长和镜头排序，但这些内容不写进正式 Seedance2.0 中文镜头正文。

## 1. 短剧剧本分析

剧本分析只做生产判断，不写资产 Prompt，不写 Seedance2.0 Prompt。

必须输出：

- 集数 / 标题 / 场次拆分
- 剧情主线：3-5 句话
- 关键剧情节点：标明不可省略节点
- 人物出入场：谁出现、谁离开、下一组是否仍在场
- 台词与 OS 时长估算
- 动作节点：只作为分镜动作拆解，不作为资产类型
- 初步资产需求：人物 / 场景 / 道具 / 群像 / 特殊状态
- 生产风险与建议分镜策略
- 建议导入的创作身份或专业身份
- 首 3 秒 / 前 6 秒剪辑潜力分析：只判断开场候选、冲突可读性和剪辑建议，不进入正式 Prompt
- 集尾钩子与下一集承接：只作为剪辑和发布 QC 参考，不改变剧本镜头内容

剧本分析阶段要保留剧情核心、台词信息和人物关系，不把文学复述误当生产分析。

### 1.1 首 3 秒 / 前 6 秒剪辑潜力分析

本项只用于剧本分析、分镜排序、覆盖镜头池、剪辑成片和发布前 QC，不进入 Seedance2.0 中文纯 Prompt。

必须判断：

- 本集最早能被观众读懂的冲突是什么。
- 前 3 秒是否能看懂人物关系、压迫关系、危机、身份差异、误会、证据或情绪张力。
- 前 6 秒是否能建立继续观看理由。
- 是否存在适合作为开场候选的镜头：脸部反应、关键台词、道具证据、身份符号、跪拜关系、压迫构图、危险动作。
- 如果剧本开场较慢，只提出剪辑建议或覆盖镜头建议，不直接改写正式分镜 Prompt。

输出模板：

```text
【首 3 秒 / 前 6 秒剪辑潜力分析】
首帧候选：
首 3 秒候选信息：
前 6 秒留存信息：
静音状态是否可读：
需要补充的覆盖镜头：
不得进入正式 Prompt 的策略词：
```

## 2. 项目生产规划

项目生产规划在剧本分析之后建立，用剧本分析结果确定生产范围。

必须建立项目控制面板：

- 项目名称
- 项目类型
- 剧集规模
- 当前集数 / 场次
- 单集目标时长
- 画幅：9:16 / 16:9 / 21:9
- 生图平台
- 生视频平台
- 当前规则版本：V8.3
- 已有角色设定、场景图、道具图、参考图
- 本次任务范围：整剧规划 / 单集分析 / 单集资产 / 分镜表 / Seedance2.0 Prompt / 覆盖镜头池 / 返工

平台与模型登记默认拆分：

```text
【资产生成模型登记】
资产类型：人物 / 场景 / 道具 / 特殊状态
首选平台：
正式模型名：
入口：ChatGPT / API / Gemini / AI Studio / Midjourney / 其他
版本：
用途：定妆 / 状态图 / 场景母图 / 道具图 / 特殊状态
输出比例：
输出分辨率：
是否可重复调用：
风险备注：

【生视频模型登记】
生视频平台：Seedance2.0
入口：Dreamina / ModelArk / Runway / 合作接口 / 其他
版本：
画幅：9:16
平台可选时长：
分辨率选项：
可用参考类型：文本 / 图片 / 视频 / 音频
默认音频模式：静音画面 / 环境声 / 对白同步 / 音频参考
风险备注：
```

若用户尚未提供完整剧本，只能建立临时项目规划；不得把临时规划当最终资产拆解依据。

竖屏项目必须额外确认：

- 核心爽点：身份反转 / 情绪爆点 / 权力压迫 / 甜虐关系 / 悬疑信息 / 复仇打脸 / 其他
- 主要人物关系：主角、对手、情感关系、权力关系、隐藏身份关系
- 竖屏观看重点：脸部、眼神、台词、道具、跪拜、身份符号、压迫构图
- 镜头节奏规则：单镜头时长偏快 / 常规 / 情绪段落可拉长
- 画面清洁度：背景是否克制、装饰是否抢主体、道具是否清晰可读
- 竖屏禁止项：人物太小、脸部不可辨、道具太小、背景抢人物、横向空间过空
- 首 3 秒 / 前 6 秒剪辑策略：只用于剪辑排序、覆盖镜头和成片 QC，不进入正式 Prompt
- 发布平台安全区：抖音 / TikTok / Reels / Shorts / 其他，根据实际平台另行检查

核心爽点、首 3 秒钩子、前 6 秒留存和集尾钩子只作为剧本分析、分镜排序、覆盖镜头池和剪辑成片参考，不作为 Seedance2.0 镜头正文描述词。

竖屏影调控制表必须在项目生产规划或风格锁定阶段建立：

```text
【竖屏影调控制表】
明度：
人物脸部可辨策略：
对比：
饱和度：
色温：
黑位：
高光：
肤色策略：
服装材质策略：
背景明暗策略：
空气感：
禁用影调：
```

“人物脸部可辨策略”必须服从真实现场拍摄光影融合原则，不能写成自动补光、独立柔光、均匀亮脸或 HDR 提亮暗部。

## 3. 风格锁定

人物、场景、道具必须分开锁定，三类风格词不能混用。

竖屏项目在风格锁定阶段必须建立“竖屏风格圣经”。风格圣经不是新增工作流阶段，而是风格锁定阶段的统一输出物。

竖屏风格圣经必须包含：

- 项目总风格
- 核心爽点和人物关系
- 竖屏影调控制表
- 色彩系统
- 光影系统
- 竖屏构图系统
- 人物造型系统
- 场景设计系统
- 道具设计系统
- 分镜镜头语言规则
- Seedance2.0 参考使用规则
- 光影融合规则
- 禁止项总表
- 质检标准

风格圣经只进入规划层、资产 Prompt、策略层和检查层，不直接写进 Seedance2.0 中文镜头正文。

### 3.1 人物风格锁定

人物风格词只管：

- 年龄感
- 身份阶层
- 面部气质
- 体态
- 发型
- 服装结构
- 服装材质
- 妆容
- 首饰
- 表演气质
- 人物连续性锚点
- 人物负面约束

### 3.2 场景风格锁定

场景风格词只管：

- 建筑类型
- 空间结构
- 入口出口
- 主位与客位
- 可走位区域
- 正向 / 反打 / 侧视 / 门口 / 窗侧等方向关系
- 墙面 / 门窗 / 地面 / 天花材质
- 时代质感
- 色彩系统
- 合乎场景逻辑的实景光源
- 空间负面约束

场景必须明确哪些方向能执行入场、离场、开门、出门、窗侧回望、主位压迫、座椅边走位等动作。

### 3.3 道具风格锁定

道具风格词只管：

- 道具名称
- 用途
- 尺寸比例
- 材质
- 纹样
- 结构
- 开合状态
- 可交互部件
- 磨损程度
- 展示角度
- 比例参考
- 关键细节
- 道具负面约束

## 4. 资产拆解

资产类型只允许：

- Character
- Scene
- Prop
- Crowd
- SpecialState

竖屏短剧中的特殊生物、特殊妆造、受伤状态、幻觉状态、变身状态等，不新增资产类型；按具体用途归入 Character、Prop 或 SpecialState。

禁止把动作关系作为资产类型。掌掴、搀扶、背人、献图、揭面、拔匕、牵袖、对峙、追赶等只进入动作节点、分镜动作拆解或站位参考说明。

资产编号格式：

```text
{集数三位}-{场次两位}-{资产类型英文}-{序号两位}
```

示例：

```text
046-01-Scene-01
046-01-Character-01
046-01-Prop-01
```

资产优先级：

- 必需：没有就无法制作关键剧情或关键镜头
- 强建议：有助于稳定画面、人物连续性或空间关系
- 可选：可替代，但会降低精度
- 氛围补充：提升质感，但不是剧情刚需

资产清单必须包含：

- 资产编号
- 资产名称
- 资产类型
- 所属集数 / 场次
- 优先级
- 用途
- 需要视图
- Prompt 平台版本
- 风险备注

场景资产必须标注方向用途，例如正向空间、反打门口方向、窗侧方向、主位方向、可走位区域。

## 5. 资产生图 Prompt

资产生图 Prompt 默认交付为 DOCX；如用户只要求文字，可先输出文本版结构。正式资产 Prompt 不使用表格，按资产编号分节，方便复制。

资产 Prompt 可以使用创作身份做前置分析，但最终可复制 Prompt 必须写具体设计规格，不写“以某某身份设计”。

### 5.1 人物资产 Prompt

人物资产必须包含：

- 场景中景光影图：用于观察人物进入真实场景后的光影、服装材质受光、肤色与场景色温关系。
- 白底三视图 + 近景定妆组合图：用于锁定脸、发型、妆容、首饰、服装领口和整体连续性。

竖屏人物资产必须额外锁定：

- 人物身份
- 年龄段
- 身形比例
- 脸部识别点
- 眼神气质
- 发型和头饰
- 服装结构、材质、主色、辅色、强调色
- 身份符号：玉佩、发冠、面具、令牌、疤痕、首饰、纹样等
- 表演气质
- 竖屏近景适配：脸部、眼神、嘴型、领口、头饰在中近景和近景中稳定可辨
- 项目风格继承
- 影调继承
- 基础光影
- 禁止项：改变脸型、发型、服装主色、身份符号、过度美颜、塑料皮肤、风格跑偏

竖屏短剧中，主角、反派、关键配角必须优先做高质量人物资产。人物资产优先级通常高于场景氛围资产。

### 5.2 场景资产 Prompt

重要场景资产必须包含：

- 顶视图
- 正视图
- 右侧视图
- 左侧视图
- 背视图

场景 Prompt 必须明确：

- 入口出口
- 主位与客位
- 家具位置
- 可走位区域
- 不可穿越区域
- 实景光源
- 正向 / 反打 / 侧视 / 门口 / 窗侧等方向用途

竖屏场景资产必须额外明确：

- 人物站位区
- 中近景拍摄区
- 可拍摄近景背景
- 前景遮挡元素：门框、帘幕、柱子、窗格等
- 中景人物活动区
- 后景识别元素
- 主光来源和主光方向
- 背景是否会抢人物
- 竖屏中上区域是否适合人物脸部和眼神
- 竖屏中下区域是否适合手部动作和道具

场景不能只做“大而全”。竖屏场景必须提供可执行的人物近景背景和可走位区域。

### 5.3 道具资产 Prompt

道具资产必须包含：

- 比例参考
- 正面 / 侧面 / 背面
- 关键细节特写
- 材质、结构、纹样、可交互部件
- 开合状态或可交互状态

竖屏道具资产必须额外锁定：

- 剧情用途：身份揭露、权力象征、证据、信物、武器、毒药、婚书、令牌、面具、玉佩、密信、血书、契约等
- 尺寸比例和参照物
- 识别锚点：纹样、符号、特殊形状、固定结构
- 竖屏近景可读性：中下区域可清楚辨认，不得过小、过暗或细节混乱
- 使用痕迹
- 光影反应：边缘高光、材质反光、暗部层次
- 禁止项：改变比例、改变主色、添加额外结构、塑料感、风格跑偏

### 5.4 特殊生物与 SpecialState 资产 Prompt

如项目存在特殊生物、怪物、神兽、幻觉形态、受伤妆造、病弱状态、失血状态、法术状态等，应按剧情用途归入 Character、Prop 或 SpecialState。

特殊生物 / SpecialState 资产必须明确：

- 类型和世界观归属
- 体型比例和尺寸参照
- 基础结构：双足、四足、多足、鱼形、鸟形、虫形、漂浮形等
- 头部结构、眼睛数量和形态
- 四肢、翅膀、尾巴、触须或特殊结构
- 关节运动逻辑
- 表皮 / 服装 / 妆造材质
- 颜色系统
- 识别锚点
- 竖屏近景可读性：头部、眼睛、轮廓和关键状态清楚
- 禁止项：结构变形、随机花纹、卡通化、风格跑偏、复杂到手机屏幕不可读

## 6. 分镜表

分镜表是剧本到 Seedance2.0 Prompt 的桥梁，不是文学复述。

必须包含：

- 镜头组编号
- 时间码
- 剧情节点
- 场景资产
- 入镜人物
- 核心镜头
- 台词 / OS
- 情绪与表演依据
- 光影设计
- 主镜头组总时长
- 风险
- 覆盖镜头池建议
- 剪辑功能：主线叙事 / 反应 / 道具证据 / 身份揭示 / 口型遮盖 / 动作桥接 / 转场 / 集尾钩子
- 是否可作为首 3 秒候选：是 / 否 / 仅作剪辑参考
- 是否可作为前 6 秒留存镜头：是 / 否 / 仅作剪辑参考
- 成片使用建议：不改变正式 Prompt 内容，只作为剪辑层参考

硬性规则：

- 新场景必须完成空间建立，建立方式由剧情功能决定，可以是全景、空镜、局部、入场动作、声音先导、遮挡揭示或后续镜头补全。
- 如果新场景建立是剧情理解必需镜头，放进主镜头组；如果只是备用剪辑素材，放进覆盖镜头池。
- 每个主镜头只保留一个主动作、一个主空间关系、一个主表演点。
- 每一组必须在本组内建立清楚的人物空间关系，但不强制第一镜完整锚定。
- 第一镜是否锚定，由剧情功能决定；后续镜头只在人物位置、朝向、距离或关键道具状态变化时补充锚定。
- 景别、机位、运镜必须有目的，不能整集重复固定中景。
- 台词和剧情核心不能随意改写。
- 情绪表达必须来自剧本文本、上下文关系、台词语气或前后动作。
- 反打镜头必须写清可见神态或可听语气，不只写“某人看着某人”。
- 光影设计必须绑定具体实景光源，不能只写空泛风格词。
- 每一组必须给出主镜头组总时长；镜头时长精确到 1 秒或 0.5 秒，不使用 0.1 秒。
- 每一组必须判断覆盖镜头池；如果没有覆盖镜头，也要标注“无覆盖镜头”并说明原因。

## 7. Seedance2.0 分镜 Prompt

Seedance2.0 分镜 Prompt 阶段包含：本场执行前置条件、本组镜头策略、本组光影继承、每组独立可复制单文本框 Prompt、证据式二次规则检查。它们不再作为独立工作流阶段拆出。

Seedance2.0 按“每组独立可复制单文本框”输出。一次只生成一组 15 秒以内主镜头；每组不能依赖上一组记忆，必须在本组内重新写清参考素材用途、资产锚定、光影继承、人物场景融合、镜头正文、音频同步模式和生成约束。

### 7.1 本场执行前置条件

正式写每组 Seedance2.0 Prompt 前，必须先做本场执行前置条件。该层只进入策略和检查，不进入中文纯 Prompt。

本场执行前置条件必须回答：

- 场面类型：静态落点 / 调度对话 / 动作场 / 情绪转折 / 信息揭示。
- 表演调度：是否需要人物走位、半步靠近或后退、坐站变化、身体转向、距离变化、视线变化。
- 镜头流动：本组是否安排轻微推近、横移、跟随、转身承接或动作中建立空间；如果全固定，必须写明剧情理由。
- 剪辑思路：双人对话是否有说话人、听者反打、切回、双人关系镜头或空间关系镜头的交替。
- 场景资产方向：当前动作和视线是否需要切换正向、反打、侧视、门口方向、窗侧方向等场景资产。
- 可走位区域：人物只能沿场景内可走位区域移动，不得穿过家具、墙、柱子、屏风、桌案或不可通行区域。
- 口型优先级：口型同步保留，但当口型不是本组主要风险时，不能因此锁死全部镜头。
- 景别与道具：近景不写当前景别不可见的空间道具；全景或中景用于锚定家具、门窗、地毯、座椅、小几等空间道具。
- 时长纪律：每组主镜头总时长控制在 15 秒以内；超过 15 秒应拆组。

除非剧情功能明确支持固定镜头，例如极短台词、静默落点、强压迫停顿、关键信息凝固、口型实测仍不稳定，否则不能把口型安全当成理由，把整组写成站桩、全固定机位和反打近景。

### 7.2 本组镜头策略

本组镜头策略必须回答：

- 戏剧目的：本组让观众获得什么剧情信息、情绪信息或动作信息。
- 信息揭示：哪些信息现在必须给，哪些信息可以延迟、遮挡或通过后续镜头揭示。
- 空间关系：第一镜锚定、动作中建立、遮挡后揭示、声音先导、第二 / 第三镜补清。
- 表演重点：台词、沉默、反应、动作停顿、眼神、手部、身体距离。
- 运镜策略：本组哪个镜头承担画面流动，哪些镜头固定；长台词镜头是否需要固定或轻微推近；短动作、视线变化或距离变化镜头是否承担运镜。
- 光影策略：本组继承哪一个场景资产的现场光源关系，人物与场景如何融合，是否需要某个关键镜头补一句可见光影结果。
- 声音策略：开口台词、画外 OS、环境声、静默或口型遮盖风险。
- 覆盖镜头需求：是否需要空间修复、反应、道具插入、情绪空镜、口型遮盖或动作桥接。
- 风险：资产锚定、站位、时长、纯 Prompt 污染、特效触发词、人物突然出现、场景混用。

策略层允许做合理导演调度推理。中文纯 Prompt 只写当前可见动作或当前可见结果，不写原因解释。

### 7.3 Seedance2.0 单文本框结构

Seedance2.0 界面只有一个文本框时，可复制 Prompt 按以下顺序组织：

```text
参考素材用途与资产锚定：
角色镜头使用名=@完整角色资产登记名，参考脸、发型、服装结构、身份符号和身形比例。
服装状态镜头使用名=@完整服装状态资产登记名，参考当前湿衣 / 战损 / 染血 / 无道具状态。
场景镜头使用名=@完整场景资产登记名，参考本镜头需要的空间结构、透视、材质、入口出口、主光方向和基础光源逻辑。
道具镜头使用名=@完整道具资产登记名，参考外观、数量、尺寸比例、材质、持握方式和剧情识别锚点。
尾帧参考镜头使用名=@完整尾帧参考登记名，参考站位、动作连续性和道具状态，不继承景别、场景、服装和光影。
视频参考镜头使用名=@完整视频参考登记名，参考镜头运动或动作节奏，不继承人物、场景、服装和光影。

全局风格与光影基准：
……

本组光影继承与人物场景融合：
……

镜头组正文内容：
镜头一……
镜头二……

音频同步模式：
音频同步……

生成约束 / Negative Prompt：
……
```

参考素材用途与资产锚定、全局风格与光影基准、本组光影继承与人物场景融合，均放在镜头正文前面。生成约束 / Negative Prompt 固定放在文本框末尾，不提前混入镜头正文。

每组只放本组会出现、会发声，或承担本组场景光影基准的资产；不得把全场资产全部塞进每一组。每组独立文本框不是全场资产清单，而是本组可见事实、可听声音和必要参考素材的最小闭环。

参考素材用途必须在多参考输入时写清楚。不得使用 P0 / P1 / P2，也不得写“主控参考 / 辅助参考 / 局部参考”标签；直接写每个素材参考什么内容。只有多参考素材之间可能发生误继承时，才写“禁止继承”和“冲突时以谁为准”。

多个参考素材冲突时，必须写清“冲突时以谁为准”。视频参考、尾帧参考、其它剧集素材、错误场景、错误服装、错误光影、错误景别等，必须写清禁止继承内容，避免模型继承错误人物、现代服装、错误场景、错误光影或错误景别。纯色底人物资产不需要单独写“不继承白底背景、白底光影和站立姿态”，除非该素材已在实测中触发误继承。

等号右侧的资产登记名必须保持用户给定的完整 token 或完整文件名，不得拆开、改名或缩写。

资产锚定必须使用：

```text
镜头使用名=@资产登记名，参考素材用途。
```

等号前是镜头正文调用名，等号后是资产名或文件名。镜头正文必须使用等号前的镜头使用名，不得混用资产文件名、未锚定别名或描述名。

```text
秦琥=@马上秦琥53.png，参考秦琥的脸、发型、服装结构、身份符号和身形比例。正文只能写“秦琥”，不能写“马上秦琥53”。
北地旌旗=@旌旗.png，参考旗面外观、数量、尺寸比例、材质和剧情识别锚点。正文写“北地旌旗”；“黑金旗面”只能作为细节描述，不能替代资产名。
```

镜头正文第一场景名必须命中等号前名称，不能自由改写成未锚定描述。

场景参考必须按镜头选择。每个镜头只使用一个正确的场景资产参考；同一组内如果出现反打、门口、窗侧、主位、外院方向、座椅边走位等镜头切换，必须同步切换到正确方向的场景资产，不能把一个场景资产写成“本组唯一场景参考”后硬套所有镜头。

生成约束 / Negative Prompt 固定写法：

```text
不要字幕，不要音乐。画面保持自然低照度，光源只来自场景内可见或剧情明确的实景光源。人物不被单独补光，暗部不被 HDR 提亮，不出现无来源轮廓光，不出现过度特效化，不出现光源方向混乱。
```

音频模式与约束必须选择一种：

```text
音频模式A：静音画面。Seedance2.0 只生成画面，不生成对白、音乐和复杂音效。
音频模式B：环境声。生成自然环境声和动作声，不生成音乐。
音频模式C：对白同步。生成可见口型对白，必须写说话人、台词、语气、音量、口型可见性。
音频模式D：音频参考。使用 @音频参考，只参考节奏、音色或口型，不改变画面内容。
```

音频约束可写：

```text
only character dialogue, lip sync, and natural ambient sound. no background music.
```

首 3 秒钩子、前 6 秒留存、爽点、爆点、付费点、剪辑策略、观众反应预测等内容，不得进入 Seedance2.0 中文镜头正文。

### 7.4 本组光影继承

每组 Seedance2.0 Prompt 必须在主镜头组前写一次“本组光影继承”。它不逐镜重复，也不只放在整集全局里。

本组光影继承只负责人物与场景融合，不写光影禁止项；光影禁止项统一放入“生成约束 / Negative Prompt”。

推荐结构：

```text
本组光影继承：
光影继承：本组继承【场景资产使用名】的现场光源关系，保持主光来源、主光方向、色温关系、阴影落点和暗部层次。人物、服装、道具、地面和背景共享同一套光源。
人物与场景融合：人物脸部受光方向必须与场景主光一致。受光面保留眼神、嘴型和面部轮廓可辨；背光面按现场光自然落暗，保留层次，不单独提亮。人物边缘光只来自场景内已有光源或明确可见光源。
局部光影变化：本组只允许【门开光线变化 / 烛火跳动 / 窗光变化 / 道具反光 / 无】。局部变化必须来自画面内可见或已建立的光源。
```

“可辨”指真实拍摄曝光关系下仍能读出眼神、嘴型和面部轮廓；不等于把暗部提亮成均匀亮脸，也不等于给人物额外补独立柔光。

镜头正文只写当前可见光影结果，例如：

```text
窗侧冷光落在角色左侧脸和肩线，右侧脸自然落暗，眼神仍能看清。
烛火在桌案边轻微跳动，暖光只扫过角色下颌和袖口。
```

### 7.5 主镜头写作规则

每个镜头写中文纯 Prompt 前，必须先判断：

1. 当前镜头使用的场景背景资产和资产方向。
2. 当前镜头可见人物。
3. 当前镜头可见道具。
4. 当前镜头的主空间关系。
5. 当前镜头唯一主动作。
6. 当前镜头是开口台词、画外 OS，还是无声音。
7. 当前镜头是否需要一个局部光影结果。
8. 当前镜头是否需要一个氛围点。
9. 当前景别是否能看见写入的家具、门窗、地毯、座椅、小几、灯架等空间道具。
10. 当前镜头是否承担表演调度、距离变化、视线变化、轻微运镜或剪辑切换。
11. 删除所有解释性、推理性、规则性、资产说明性语言。
12. 只保留观众能看见和听见的内容。

镜头正文推荐句式：

```text
镜头一，持续5秒。场景镜头使用名，景别，机位 / 运镜。
本镜开始时：角色A……，角色B……，道具……。
核心动作过程：角色A完成一个主动作……，可见表情 / 视线 / 身体反应……，道具或环境出现……物理反馈。
镜头结尾停在：角色A……，角色B……，道具……，视线……。
音频同步：角色A使用参考音频编号，声音状态，说出“台词”。
```

人物表演镜 / 动作镜 / 对白镜正文顺序：主体 → 当前状态 → 核心动作 → 场景关系 → 可见表演 → 光影物理 → 运镜 → 尾帧。

空间建立镜 / 氛围镜正文顺序：场景空间 → 光源与环境物理 → 主体位置或主体缺席 → 运镜 → 尾帧。

运镜主导镜 / 转场镜正文顺序：镜头运动 → 掠过或揭示的主体 / 道具 / 空间 → 光影物理 → 结尾落点。

参考图生视频 / 尾帧承接镜单文本框顺序：参考素材用途区 → 本镜开始状态 → 核心动作 → 运镜变化 → 尾帧状态。参考素材用途进入 Seedance2.0 单文本框 Prompt，用来告诉模型素材怎么用；中文镜头正文仍只写当前可见、可听内容。

人物动作必须写成可见身体动作，优先按：整体姿态 → 头部动作 → 视线 / 表情 → 手部动作 → 道具或环境接触。

抽象情绪词不得单独作为控制指令。“害怕、愤怒、犹豫、崩溃、坚定”等必须转写为肩背、呼吸、头部、视线、眉眼、嘴唇、手指、道具接触和声音状态。

每个 S/A 级关键镜头允许写 2-3 个物理因果细节；B 级最多写 1-2 个；C 级只写一个视觉重点。物理细节必须服务主体动作、道具、光源或环境接触，不堆叠无关雨、雾、风、火、水花、光斑和尘埃。

无台词镜头不写音频同步。开口台词镜头必须写“开口说话，口型跟随参考音频”或等价描述。

内心 OS 镜头必须在画面层明确角色不张嘴、嘴唇保持闭合；音频同步层必须明确“OS为画外内心声”，尽量使用“响起”而不是“说出”。

### 7.6 空间、站位与场景资产方向

视频生成模型不能被假设会可靠继承上一组的空间、站位、人物朝向、距离关系、动作尾帧或手持道具状态。每一组必须在本组内建立观众能理解的人物空间关系。

镜头组之间需要连续性锚定。下一组第一个镜头必须锚定上一组结束镜的人物位置、动作关系和道具状态，但必须重新设计景别、机位或调度入口，不能机械延续上一组景别。

如果第一镜是对话、对峙、多人物同场、近距离动作、递物、拉扯、持刀、人物站位影响台词理解，应做轻量站位锚定。

如果第一镜是新场景建立、空镜、入场、走动、追赶、闪避、突发动作、新人物首次出现、悬念揭示、遮挡构图、先声后人，可以通过动作、遮挡、局部、声音或后续镜头建立关系。

轻量站位锚定至少包含：

- 场景镜头使用名
- 主要人物位置
- 人物朝向和相对关系
- 人物距离
- 关键道具或状态

场景资产方向必须服务当前动作。门口离场、入门、外院方向、窗侧回望、主位压迫、座椅边走位等动作，必须选择真实包含该方向的场景资产。禁止在不含门、不含窗、不含可走路径的视角里硬写出门、进门、窗边或穿越家具的动作。

近景只写当前近景中可见的人脸、肩颈、手部、局部道具或局部环境。近景中看不见的小几、地毯、门口、座椅整体、灯架整体，不得作为空间锚定写入。

### 7.7 指令负载与运镜

15 秒镜头组内，每组优先保证：

1. 镜头结构
2. 人物动作
3. 站位关系
4. 台词与口型同步
5. 表演调度和剪辑流动

推荐承载：

- 单镜头长镜头：最稳，可承载 8-12 条核心指令。
- 2-3 个镜头：短剧剧情对话最推荐，每镜 5-8 条核心指令。
- 4 个镜头：可做但要压缩动作，每镜 4-6 条核心指令。
- 5-6 个镜头：高风险，只适合简单覆盖镜或快速反应镜，每镜 3-5 条核心指令。
- 6 个以上镜头：不建议放进同一个 15 秒生成，应拆成下一组。

每镜约束：

- 景别：每镜 1 个
- 运镜：每镜 0-1 个
- 主体人物：1-3 人最稳
- 人物站位：每镜 1 组主关系
- 主动作：每镜 1 个，复杂动作拆节点
- 表情 / 情绪：每镜 1 个主表演点
- 说话 / 台词：每镜 1 个说话人最稳
- 景深：每镜最多 1 个
- 光影：全组 1-2 条，全局统一
- 氛围点：每镜最多 1 个

运镜纪律：

- 每个镜头最多 1 个运镜。
- 每组原则上安排 1 个镜头承担画面流动，例如轻微推近、横移、跟随、转身承接、半步走位或动作中建立空间。
- 每组通常只给 1-2 个镜头安排轻微运镜；超过 2 个运镜时必须有动作或剪辑理由，并警惕过载。
- 固定镜头不是默认选择，只在长台词、静默落点、强压迫停顿、空间锚定、口型实测不稳或情绪需要压住时优先固定。
- 长台词镜头可以固定以保证口型、表演和台词同步；若口型风险降低，可加入一个轻微推近、横移或跟随，但不能同时加入复杂走位。
- 双人对话不默认站桩反打近景，必须判断说话人、听者反打、切回、双人关系镜头、空间关系镜头和场景资产方向的剪辑组合。
- 人物走位必须沿场景可走区域完成，不得穿过家具；走位和运镜不能同时复杂化。

### 7.8 语言情绪、表演节奏与高风险词

语言情绪是声音层指令，写在音频同步里。表演节奏是画面层指令，写在镜头正文里。

语言情绪必须绑定可听声音状态。抽象情绪词只能作为辅助，不得单独作为控制指令。

推荐拆分：

```text
角色肩背微微僵住，嘴唇短促张开，开口说话，口型跟随参考音频。
音频同步：角色使用参考音频，声音短促，音量压低，尾音绷紧，说出“台词”。
```

避免把容易触发突发视觉特效的词写进中文纯 Prompt，除非项目明确就是奇幻、法术、超自然或特效风格，且用户确认需要。

高风险词类型：

- 面色突变类：脸色惨白、脸色刷白、面色煞白、血色褪尽、突然变白、面如死灰。
- 发光发亮类：眼睛发光、瞳孔亮起、身上发光、周身泛光、金光闪过。
- 气场外放类：杀气四溢、气场爆开、威压扩散、压迫感弥漫、寒意蔓延、黑气缠绕。
- 粒子烟雾类：黑雾升起、白雾缠绕、烟气从身上冒出、空气扭曲、尘埃炸开。
- 突然变形类：表情瞬间崩裂、眼神瞬间变红、面部扭曲、身体僵成雕像。
- 抽象情绪视觉化类：恐惧笼罩全身、心碎感扩散、怒意燃烧。
- 镜头特效类：画面炸裂、画面震碎、时间静止、世界凝固、空间扭曲。
- 呼吸白气类：轻吐一口气、吐出一口气、哈出一口气、白气从口中散开。

替代表达优先写可见表演、动作停顿、声音状态或光影结果。

呼吸白气只有在明确冬天、寒夜、雪地、严寒室外等环境中才可写。

### 7.9 V8.3 证据式二次规则检查

正式 Seedance2.0 分镜 Prompt 生成后必须执行二次规则检查。检查必须分为：

- A. 格式硬检
- B. 导演连续性实检
- C. 修正版

草稿版可以简查，但正式复制版必须完整检查并修正后再交付。用户明确说“先别检查，只出草稿”时可以跳过，但必须标注“未执行 V8.3.1 二次检查”。

A. 格式硬检必须检查：

- 是否输出本场执行前置条件和本组镜头策略。
- 是否在主镜头组前输出本组光影继承。
- 场景资产锚定是否完整。
- 每个镜头第一场景名是否命中场景资产锚定等号前名称。
- 纯 Prompt 是否出现解释、推理、规则提醒、资产用途说明或污染词。
- 策略层 / 检查层的导演判断是否没有误写进纯 Prompt。
- 每组是否有主镜头组总时长。
- 单镜头时长是否只到 1 秒或 0.5 秒。
- 覆盖镜头池是否与主镜头组分开。
- 覆盖镜头是否逐条检查并满足完整结构：编号、持续时间、场景资产使用名、景别、机位 / 运镜、主体、主动作、声音状态。
- 开口台词镜头是否有音频同步和口型跟随参考音频。
- 内心 OS 镜头是否明确不张嘴、嘴唇保持闭合。
- 生成约束是否使用 V8.3.1 固定写法，且字幕、音乐和光影禁止项没有写进镜头正文。
- 是否出现高风险特效触发词。
- 是否每镜最多 1 个运镜。
- 15 秒内镜头数量、动作、站位、表演、光影、氛围是否过载。

B. 导演连续性实检必须逐镜回答：

- 上一镜状态：谁在什么位置、什么姿态、手里有什么道具。
- 本镜状态：是否突然换位、坐下、站起、回头、转身、拿起或放下道具。
- 动作动词：是否符合人物姿态和坐站高低关系。
- 视线方向：是否有空间依据。
- 道具状态：是否莫名出现、消失、换手、离开原位或状态突变。
- 景别可见性：动作和道具是否能在当前景别、遮挡关系和机位中看见。
- 台词 / 动作控制：台词镜头是否被复杂动作或过多运镜干扰。
- 镜头流动：是否不必要地退回站桩、全固定机位、连续反打近景。
- 场景资产方向：是否存在一个场景资产用到底造成门口、窗侧、主位或外院方向错误。
- 可走位区域：人物是否穿过家具、墙、柱子、屏风、桌案或不可通行区域。

连续性实检发现语义矛盾时，即使格式硬检全部通过，也必须判为不通过并输出修正版。

二次检查输出格式：

```text
【V8.3 证据式二次规则检查】
结论：通过 / 需要修正

A. 格式硬检
1. 检查项：
   结果：通过 / 不通过
   证据：
   修正：

B. 导演连续性实检
镜头一连续性检查：
上一镜状态：
本镜状态：
动作动词：
视线方向：
道具状态：
景别可见性：
台词 / 动作控制：
镜头流动：
场景资产方向：
可走位区域：
结果：通过 / 不通过
证据：
修正：

C. 修正版
如有问题，直接输出完整修正版。

处理：
已输出修正版 / V8.3.1 检查通过 / 未执行 V8.3.1 二次检查
```

### 7.10 竖屏构图与节奏专项规则

当项目为 9:16 竖屏时，本规则进入本场执行前置条件、本组镜头策略、分镜 Prompt 检查层和终检层；不得把说明性规则原样写进中文镜头正文。

竖屏画面默认分为三段：

- 上区：头部空间、权力压迫、牌匾、窗光、建筑高度。
- 中区：人物脸部、眼神、表情、核心冲突。
- 下区：手部动作、道具、服装层次、跪拜关系。

竖屏视觉重点通常在：

- 中上区域：人物脸、眼神、表情。
- 中下区域：手部动作、道具、跪拜、身份符号。

竖屏常用景别：

- 面部特写：用于眼神、身份反转、情绪爆点。
- 中近景：人物从头到胸口或腰部入画，适合台词和情绪。
- 中景：人物从头到膝上入画，适合站位、道具和轻微走位。
- 全身镜头：只在身份、跪拜、冲突站位需要时使用。
- 大全景：谨慎使用，必须服务空间关系，且人物关系仍要清楚。

竖屏构图方式：

- 中心压迫构图：主角或权力人物居中。
- 高低位构图：上位者在画面上区，下位者在画面下区。
- 前景遮挡构图：门框、帘幕、柱子、袖摆形成层次，但不得遮挡人物脸。
- 对峙构图：两人上下错位或前后错位，不建议左右过宽。
- 负空间构图：人物头顶或身后留压迫空间。
- 道具锚点构图：令牌、剑、扇、信物位于中下区，作为剧情焦点。

竖屏构图禁止项：

- 人物太小
- 人物脸部不可辨
- 主体偏到画面边缘
- 重要人物被裁掉头
- 道具位于安全区外
- 背景抢人物
- 左右空间过空
- 宽屏横向调度直接硬套到竖屏

竖屏发布安全区：

- 竖屏构图分区不等于平台 UI 安全区。
- 核心脸部、眼神和嘴型优先放在画面中上安全区域。
- 核心道具可放在中下区域，但不得贴近底边、右侧边缘或平台 UI 常见遮挡区。
- 重要身份符号不放在最底部 20%。
- 关键文字不由 Seedance2.0 生成。
- 字幕、标题、集数、付费提示、互动提示必须在后期使用平台安全区模板检查。
- 同一条短剧如发布到抖音 / TikTok / Reels / Shorts，应分别检查 UI 遮挡。

竖屏成片节奏建议只用于后期剪辑与发布前 QC，不进入正式 Seedance2.0 镜头 Prompt：

- 1.5-2.5 秒：反应镜头、眼神特写、道具特写。
- 2.5-4 秒：常规台词镜头。
- 4-6 秒：情绪爆发、转身、跪拜、揭露身份。
- 6-8 秒：多人对峙、复杂调度、关键剧情反转。

正式 Seedance2.0 镜头 Prompt 仍按每镜持续时间和主镜头组总时长书写，每组主镜头总时长不超过 15 秒。

成片中不建议频繁使用超过 8 秒的单镜头段落。长镜头必须有明确情绪推进，否则容易拖节奏。

每条竖屏镜头只完成一个核心目标：

- 一个核心表情
- 一个核心动作
- 一句核心台词
- 一个核心关系变化
- 一个明确结束画面

禁止同一镜头内塞入过多台词、多次转场、多人同时复杂动作，或让镜头运动和人物动作同时过度复杂。

### 7.11 AI 真人镜头风险分级

R1 低风险：空镜、背影、手部插入、道具特写、环境变化、无口型反应镜。

R2 中风险：单人中近景、短台词、轻微转头、轻微走位、简单持物。

R3 高风险：正脸长台词、哭戏、激烈情绪、双人近距离互动、递物、牵手、跪拜、掌掴、推搡。

R4 极高风险：多人同场复杂调度、打斗、拥抱亲吻、长口型、手部复杂操作、镜头运动和人物运动同时复杂化。

处理规则：

- R1 / R2 可直接生成。
- R3 必须准备覆盖镜头。
- R4 必须拆镜，不能一条 Prompt 硬做。
- R4 如因剧情必须保留，应拆成主动作镜、反应镜、手部 / 道具特写、声音 OS 或覆盖镜头组合。

风险等级必须进入分镜表和终检，不进入 Seedance2.0 中文镜头正文。

### 7.12 台词密度规则

2.5-4 秒持续时间的台词镜头：

- 只保留一句核心台词。
- 中文建议不超过 12-18 字。
- 超过则拆成说话人镜头 + 听者反应 + OS 或下一镜。

4-6 秒持续时间的情绪台词镜头：

- 可以保留一句较完整台词。
- 不叠加复杂走位和复杂运镜。
- 允许轻微推近或固定机位保护口型。

6 秒以上持续时间的长台词镜头：

- 必须说明为何不能拆。
- 优先固定或轻微推近。
- 必须准备口型遮盖覆盖镜头。

双人对话原则：每镜一个说话人最稳。听者反应、手部、道具和空间关系可以作为覆盖镜头或反打镜头承担剪辑节奏。

### 7.13 Seedance2.0 常见失败与修法映射

```text
人物脸漂移：加强完整人物资产调用和参考素材用途，减少场景和风格词，镜头改为中近景或近景，删除无关人物描述。

人物像贴在背景上：强化本组光影继承，写清主光方向、受光侧、暗部侧，强调人物、道具、地面和背景共享同一套光源。

动作过度：缩短动作链，只保留一个核心动作，写清开始状态和结束状态。

动作太弱：增加身体部位动作，写清头部、视线、手部、道具接触和环境反馈。

手部 / 道具错乱：减少手部复杂操作，写清哪只手持有道具、道具接触哪里、是否遮挡，必要时改成道具特写或背影覆盖镜头。

光线过亮：删除“高光、轮廓、清晰、明亮”等词，改写为低照度、暗部保留、只受实景光源影响。

雨水 / 雾气过特效化：写成低亮度、局部、受风向或光源影响的物理结果，禁止发光粒子化。

镜头不按要求运动：把运镜写成单一动作，必要时拆成两个镜头或写清稳定起止状态。

口型失败：减少正面台词镜，改用侧脸、背影、听者反应、环境覆盖镜头或后期配音。

字幕 / 乱码文字：确认生成约束使用固定写法中的“不要字幕”；后期字幕单独添加。

自动生成音乐：确认音频模式不是自动配乐，生成约束使用固定写法中的“不要音乐”；后期音乐另做。
```

### 7.14 首 3 秒策略不得污染 Prompt

正式 Seedance2.0 镜头正文只按剧本内容拆分。

禁止写入：

- 首 3 秒钩子
- 前 6 秒留存
- 爽点
- 爆点
- 付费点
- 反转感
- 观众会继续看
- 信息流抓人
- 剪辑策略

如果需要为首 3 秒补镜头，只能在覆盖镜头池中写可见、可生成、符合剧本的画面，例如脸部反应、道具证据、身份符号、压迫关系、危险动作或静音可读的冲突镜头。

## 8. 覆盖镜头池

覆盖镜头池是备用剪辑素材，用于剪辑补位、情绪转场、动作强调、空间修复、节奏缓冲和口型风险规避。

覆盖镜头不并入主镜头组总时长，但每条覆盖镜头本身仍是要生成的视频镜头，必须有自己的持续时间。

覆盖镜头池必须按每一组判断和书写。整集完成后可以追加一次“全片覆盖镜头补漏”，用于检查是否缺少统一空镜、关键道具插入、转场或口型遮盖素材。

覆盖镜头池正式交付时必须单列，但输出内容里不要写“独立生成”“不计入主时长”“供后期剪辑使用”等说明性文字。

只要写“有覆盖镜头”，每条覆盖镜头都必须满足可直接生成结构：

1. 覆盖镜头编号
2. 持续时间，精确到 1 秒或 0.5 秒
3. 场景资产使用名，必须命中场景资产锚定等号前名称
4. 景别
5. 机位 / 运镜，每条最多 1 个运镜
6. 可见主体；空镜必须写清可见环境主体
7. 唯一主动作；空镜必须写清可见环境变化
8. 声音状态：无台词 / 环境声 / 可听动作声 / OS

覆盖镜头类型：

- 空间修复镜头
- 动作桥接镜头
- 反应镜头
- 道具插入镜头
- 情绪转场空镜
- 视线承接镜头
- 权力关系镜头
- 遮挡 / 隐藏信息镜头
- 节奏缓冲镜头
- 口型风险规避镜头
- 新场景建立镜头
- 离场 / 入场衔接镜头

覆盖镜头剪辑用途标签：

- 可遮挡口型失败
- 可遮挡手部失败
- 可遮挡动作断裂
- 可做情绪停顿
- 可做身份揭示前铺垫
- 可做身份揭示后反应
- 可接下一场
- 可做首 3 秒候选
- 可做前 6 秒留存辅助
- 可做付费点前停顿
- 可做集尾钩子

覆盖镜头可以服务首 3 秒剪辑，但覆盖镜头正文仍必须所见即所得，不写“钩子感、爽点感、留存感”等说明词。

覆盖镜头也要进入导演连续性实检，确认它是否接得住主镜头尾帧、人物视线、动作方向和道具状态。

如果没有覆盖镜头，也必须标注：

```text
覆盖镜头池：无。本组主镜头已完成空间建立、台词同步和动作闭合，暂不需要覆盖镜头。
```

## 9. 终检与返工

每个阶段交付前必须终检。终检不是新增创作阶段，而是交付前的质量闸门。

### 9.1 通用终检

- 是否按 V8.3.1 九阶段推进，并按用户指令确定本轮执行起点。
- 是否区分生图平台和生视频平台。
- 人物 / 场景 / 道具风格锁定词是否分离。
- 资产类型是否只包含 Character / Scene / Prop / Crowd / SpecialState。
- 动作关系是否没有被列为资产类型。
- 资产编号是否符合格式。
- 资产 Prompt 是否不使用表格。
- 场景五视图、人物两类图、道具比例参考是否完整。
- 竖屏项目是否建立核心爽点、竖屏影调控制表和竖屏风格圣经。
- 首 3 秒 / 前 6 秒规则是否只用于剧本分析、剪辑策略和成片 QC，没有进入正式 Prompt。
- 是否给出每组主镜头组总时长，且每组主镜头总时长不超过 15 秒。
- 是否建立资产生成模型登记和生视频模型登记。
- 竖屏资产 Prompt 是否补足人物脸部识别点、身份符号、场景可拍近景区、道具竖屏可读性等字段。

### 9.2 分镜与 Seedance2.0 终检

- 正式 Prompt 前是否先做本场执行前置条件和本组镜头策略。
- 主镜头组前是否写入本组光影继承。
- 本组光影继承是否只写人物与场景融合、现场光源关系和局部光影变化，没有写光影禁止项。
- 生成约束 / Negative Prompt 是否使用 V8.3.1 固定写法。
- 镜头正文是否只写当前可见光影结果，没有写光影规则说明。
- 人物、服装、道具、地面和背景是否共享同一套现场光源关系。
- 人物脸部受光方向是否与场景主光方向一致。
- 眼神、嘴型、面部轮廓是否在真实曝光关系内可辨，且没有通过独立柔光、自动补光、美容补光或 HDR 提亮暗部实现。
- 新场景是否完成空间建立，建立方式是否服务剧情。
- 每个主镜头是否只有一个主动作和一个主空间关系。
- 每组是否在本组内建立清楚的人物空间关系。
- 每组是否判断人物走位、距离变化、视线变化、轻微运镜或场景资产方向切换。
- 是否避免不必要的站桩、全固定机位和连续反打近景。
- 双人对话是否有说话人、听者反打、切回、双人关系镜头或空间关系镜头的剪辑思路。
- 如果整组全固定，是否有极短台词、静默落点、强压迫停顿、关键信息凝固或口型实测风险明确等剧情理由。
- 场景资产方向是否匹配入门、出门、窗侧、主位、座椅边、外院方向等动作和视线。
- 人物是否只沿可走位区域移动，没有穿过家具、墙、柱子、屏风、桌案或不可通行区域。
- 每镜是否接得住上一镜尾帧的人物位置、姿态、朝向、道具状态和视线关系。
- 坐站高低关系是否支持当前镜头写入的“抬眼、垂眼、低头、回头、转身”等动作动词。
- 道具是否没有莫名出现、消失、换手、移位或状态突变。
- 景别、机位、运镜是否服务剧情。
- 是否执行低风险运镜策略。
- 情绪表达是否有剧本依据。
- 覆盖镜头池是否单列，且没有写入说明性文字。
- 每一组是否给出主镜头组总时长。
- 镜头时长是否只精确到 1 秒或 0.5 秒。
- 是否执行“格式硬检 + 导演连续性实检”的两层检查。
- 是否符合所见即所得第一原则。

竖屏项目还必须检查：

- 是否符合 9:16 竖屏构图。
- 人物脸部、眼神、嘴型是否在手机屏幕上可辨。
- 人物是否太小，主体是否偏到画面边缘。
- 重要人物是否被裁掉头或被前景遮挡脸部。
- 核心冲突是否集中在中上区域，手部动作、道具、身份符号是否在中下区域可读。
- 道具是否承担剧情功能，数量、比例、颜色和材质是否稳定。
- 背景是否抢人物，左右空间是否过空。
- 人物服装、发型、头饰、身份符号是否继承资产。
- 场景结构、透视、材质、主光方向是否继承场景资产。
- 人物与背景是否共享同一套现场光源关系。
- 是否出现色彩跳变、无来源光、过度特效化。
- 镜头运动是否符合 Prompt，动作是否过度或缺失。
- 每组主镜头总时长是否不超过 15 秒；超过 15 秒是否已经拆组。
- 覆盖镜头是否不计入主镜头组总时长，但每条覆盖镜头都有自己的持续时间。
- 正式 Seedance2.0 镜头 Prompt 是否没有写入后期剪辑使用时长、可用区间或剪辑排序说明。
- AI 真人风险等级是否已判断；R3 是否准备覆盖镜头，R4 是否拆镜。
- 台词密度是否符合镜头时长，长台词是否准备口型遮盖覆盖镜头。
- 字幕和音乐是否保持生成阶段 / 后期阶段分层。
- 正式 Prompt 是否没有混入首 3 秒、前 6 秒、爽点、爆点、留存等剪辑策略词。

只要 Seedance2.0 终检任一项不合格，必须回到检查层重写，不得直接交付纯 Prompt。

### 9.3 成片剪辑与发布前 QC

只在用户提供剪辑成片、粗剪版、发布版或要求做剪辑检查时启用。本项不反向污染已经通过的 Seedance2.0 正式镜头 Prompt。

检查：

- 首帧是否能识别主体、场景或冲突。
- 首 3 秒是否出现可读的冲突、身份、危机、误会、道具证据或情绪张力。
- 前 6 秒是否建立继续观看理由。
- 字幕、人物脸、嘴型、关键道具是否避开平台 UI 遮挡。
- 音乐、环境声、台词是否没有互相压制。
- 是否存在慢入场、空镜过长、人物太小、道具不可读、台词信息过迟的问题。
- 集尾是否有明确停顿、问题、反转落点或下一集承接动作。
- 发布平台如为 Shorts，超过 1 分钟时是否单独检查音乐和 Content ID 风险。

处理：

- 只提出剪辑调整、镜头排序、覆盖镜头补充、字幕节奏或声音混音建议。
- 不为了首 3 秒重新改写已经符合剧本和连续性的正式 Seedance2.0 Prompt。
- 如需补素材，进入覆盖镜头补拍模式。

### 9.5 返工规则

返工时不要整体推翻已确认内容，除非用户明确要求。

修正 Prompt 必须只修正错误点，不能重新生成整个设定。除非用户明确要求重做，否则必须保持当前人物、场景、构图、动作、台词和已确认资产不变。

错误修正方式：

```text
重新做得更电影感一点。
```

正确修正方式：

```text
保持当前人物、场景、构图和动作不变，只修正光影融合。
人物面部受光方向必须与场景左侧主光一致，背光面自然落暗但保留层次。
眼神、嘴型和面部轮廓在真实曝光关系内可辨，禁止人物面部独立柔光，禁止自动补光，禁止 HDR 提亮暗部。
禁止改变服装颜色，禁止新增无来源强光，禁止背景变亮。
```

推荐返工方式：

```text
只返工第 X 场分镜，不动资产清单。
```

```text
只清理纯 Prompt 里的解释性文字，不改检查层。
```

```text
只把光影改成真实现场光源融合关系，不改动作和台词。
```

```text
为第 X 组补覆盖镜头池，不计入主时长。
```

如果用户给出新规则、新模板或项目内补丁，以用户最新规则为准；若新规则与 V8.3 冲突，必须指出冲突并按用户最新确认执行。

## 附录 A：V8 重构时发现的逻辑冲突与处理

1. 流程顺序冲突：V7.1 先做项目初始化，用户指定 V8 先做短剧剧本分析，再做项目生产规划。
   处理：V8 改为剧本分析优先，项目生产规划使用剧本分析结果建立；若剧本缺失，只能做临时项目规划。

2. 阶段重复冲突：V7.1 把本场执行前置条件、本组镜头策略、Seedance Prompt、覆盖镜头池、证据式检查、终检拆成多个相邻阶段，容易重复输出。
   处理：V8 将本场执行前置条件、本组镜头策略和证据式二次检查并入 Seedance2.0 分镜 Prompt 阶段；覆盖镜头池保留为独立交付阶段。

3. 覆盖镜头池位置冲突：V7.1 要求每组都判断覆盖镜头池，但用户 V8 流程把覆盖镜头池放在 Seedance2.0 分镜 Prompt 之后。
   处理：V8 规定分镜表阶段先给覆盖建议，Seedance 阶段同步判断每组是否需要覆盖，覆盖镜头池阶段正式单列交付和补漏。

4. 口型安全与表演调度冲突：V7.1 早期偏向固定镜头保护口型，后续补丁要求表演调度优先于纯静态口型保护。
   处理：V8 明确固定镜头不是默认；当口型不是主要风险时，台词镜头可加入轻微推近、横移或跟随，但不得叠加复杂走位和复杂运镜。

5. 单文本框与纯 Prompt 分层冲突：Seedance2.0 只有一个文本框，但纯 Prompt 又不能写 Negative Prompt。
   处理：V8 明确单文本框分为全局区和镜头正文区；全局区可写生成约束和 Negative Prompt，镜头正文区执行所见即所得。

6. 终检与二次检查重复冲突：V7.1 同一类检查项在证据式二次检查和终检中重复出现。
   处理：V8 将“证据式二次检查”作为 Seedance Prompt 交付前的修正机制，将“终检”作为阶段性交付闸门，只保留必要清单。

## 附录 B：V8.1 光影融合补丁说明

1. “人物脸部可读”不能被模型理解为自动补光。
   处理：V8.1 改为“眼神、嘴型、面部轮廓必须在真实曝光关系内可辨，暗部不能吞脸”；同时明确“可辨”不等于自动补光、独立柔光、均匀亮脸或 HDR 提亮暗部。

2. 光影继承的重点不是逐镜重复规则，而是每组建立人物与场景融合关系。
   处理：V8.1 要求每组 Seedance2.0 Prompt 在主镜头组前写一次“本组光影继承”，不逐镜重复，也不只放在整集全局里。

3. 光影禁止项不进入本组光影继承，也不进入中文镜头正文。
   处理：V8.1 将光影禁止项统一放入“生成约束 / Negative Prompt”。

4. 本组光影继承只写三件事：光影继承、人物与场景融合、局部光影变化。
   处理：镜头正文只写当前可见光影结果，例如窗侧冷光落在脸侧、烛火扫过袖口、背光面自然落暗。

## 附录 C：V8.2 竖屏专项融合说明

1. 竖屏规则不改变 V8.3.1 九阶段主流程。
