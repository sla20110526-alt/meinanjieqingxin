# 短剧 AI 视频生产规则 V7.1 统一版

规则状态：当前唯一活动规则源。

来源：基于《短剧剧本分析智能体_知识库与工作流手册_v1.2_修订版》、V6 纯 Prompt 修正规则、V7 统一规则，以及后续关于证据式二次检查、本组镜头策略前置层、Seedance 实测失败反馈记录的升级讨论。

适用范围：短剧剧本分析、项目规划、风格锁定、资产拆解、资产生图 Prompt、分镜表、Seedance2.0 分镜 Prompt、覆盖镜头池、终检与返工。

## 0. V7.1 升级前逻辑自查结论

V7.1 升级前已经完成一次规则逻辑自查，结论如下：

1. 字幕和音乐禁止项不能写进中文纯 Prompt，应放入全局生成约束 / Negative Prompt。
2. Seedance2.0 只有一个文本框时，全局风格、光影纪律、生成约束、音频约束和角色资产锚定应放在镜头正文前面。
3. 创作身份有用，但只用于设计任务说明、检查层、前置分析和返工判断，不进入最终纯 Prompt。
4. 语言情绪和表演节奏应分层控制：语言情绪写进音频同步，表演节奏写进可见画面。
5. 光影应采用“全局纪律 + 关键镜头局部可见结果”的方式，不在每个镜头重复完整光影规则。
6. 氛围营造建议采用两步法：第一版干净生产，第二版按关键镜头局部加氛围。
7. 覆盖镜头不并入主镜头组总时长计算，但每条覆盖镜头本身仍是要生成的视频镜头，必须有自己的持续时间，并满足可直接生成的完整镜头结构；输出覆盖镜头池时不写“独立生成”“不计入主时长”等说明性文字。
8. 旧规则存在重复承载问题，V7.1 起只把本文件作为当前活动规则源；旧文件只作为历史参考。
9. 场景资产调用必须使用“镜头使用名=@资产登记名”的锚定方式，镜头正文必须使用镜头使用名，避免模型把自由描述误认为新场景。
10. 正式 Seedance2.0 分镜 Prompt 生成后必须执行 V7.1 证据式二次规则检查；有问题直接输出修正版，无问题标注检查通过。
11. 每一组必须在本组内建立观众能理解的人物空间关系；是否在第一镜完成站位锚定，由剧情功能判断，不能机械套用。
12. 分镜 Prompt 生成前必须先做“本组镜头策略”，先判断戏剧目的、信息揭示、空间关系、表演、运镜、光影和覆盖镜头需求，再写 Prompt。
13. Seedance 实测失败必须记录失败类型、问题镜头、原 Prompt、修正版和是否沉淀规则补丁，形成反馈闭环。

## 1. 最高优先级原则

Seedance2.0 中文纯 Prompt 必须执行“所见即所得”第一原则。

纯 Prompt 只写当前镜头中观众能够直接看见、直接听见的内容。不能为了让模型理解剧情，把导演解释、剧情意图、人物心理判断、资产用途说明、空间推理过程、隐藏因果、不可见动作路径、规则提醒、生成建议写进纯 Prompt。

该原则优先级高于美术风格、镜头丰富度、运镜设计、动作完整度、台词节奏、情绪解释、资产调用说明和导演意图表达。

纯 Prompt 禁止写：

- 剧情解释
- 人物心理判断
- 导演意图
- 资产用途说明
- 画面外动作路径
- 规则提醒
- 二选一或不确定表达
- 模型理解辅助语
- 当前景别中不可见的表情、眼神、手指微动作
- “为了表现、用于表现、暗示、说明、代表、意味着、基于参考图、不要、避免、可以、或者、必要时”等污染词

这些内容可以进入检查层、设计任务说明或生成约束，但不能进入中文纯 Prompt。V7.1 禁止的是把逻辑推理写进纯 Prompt，不禁止在策略层 / 检查层做导演判断、连续性判断、表演合理性判断和空间调度判断。

## 2. 固定工作流

除非用户明确要求跳过，否则按以下顺序推进：

1. 项目初始化
2. 剧本生产分析
3. 风格锁定
4. 资产拆解
5. 资产生图 Prompt
6. 分镜表
7. 本组镜头策略
8. Seedance2.0 分镜 Prompt
9. 覆盖镜头池
10. 证据式二次规则检查
11. 终检与规则沉淀

不得直接从剧本跳到最终 Seedance2.0 Prompt。每个阶段都要产出可检查的中间结果。

如果用户要求跳过阶段，应先提醒风险；若用户仍要求跳过，则执行，并标注跳过造成的风险。

## 3. 创作身份与设计任务说明

每一步都可以先做“设计任务说明”，用创作身份完成前置分析。创作身份负责判断，不进入最终纯 Prompt。

常用影视生产身份：

- 导演：判断剧情重点、表演强度、情绪节奏、戏剧目的。
- 摄影指导：判断景别、焦段、机位、运镜、构图和剪辑节奏。
- 灯光指导：判断光源方向、明暗比例、是否允许补光、阴影层次。
- 美术指导：判断场景、服装、道具、材质、色彩和时代质感。
- 动作指导：拆解主动作、接触关系、走位、动作节点和安全距离。
- 声音指导：判断台词、语气、口型同步、环境声和音乐禁用。
- Prompt 工程架构师：把以上判断压缩成可见可听、低污染、可执行的 Prompt。
- AI 分镜统筹：检查资产调用、人物出入场、镜头连续性、覆盖镜头和时长。

非影视项目可按内容导入对应专家身份，例如：中国历史学家、美国历史学家、经济分析师、理论物理学家、医学顾问、法律顾问等。专家身份只用于前置判断，不进入最终纯 Prompt。

示例：

```text
设计任务说明：
请以导演、摄影指导、灯光指导、动作指导、美术指导、声音指导、Prompt 工程架构师的身份，先判断本镜头的剧情重点、表演节奏、机位、光源、动作节点和音频同步风险。身份判断只进入检查层，不进入中文纯 Prompt。
```

## 4. 项目初始化规则

开始新项目或新剧集时，先建立项目控制面板。必须确认或记录：

- 项目名称
- 项目类型
- 剧集规模
- 单集目标时长
- 画幅：9:16 / 16:9 / 21:9
- 生图平台
- 生视频平台
- 当前规则版本：V7
- 已有角色设定、场景图、道具图、参考图
- 本次任务范围：整剧规划 / 单集分析 / 单集资产 / 分镜表 / Seedance2.0 Prompt / 覆盖镜头池

平台默认拆分：

- 人物资产：GPT Image2 / Nano Banana2
- 场景资产：GPT Image2 / Midjourney / Nano Banana2
- 道具资产：GPT Image2 / Midjourney / Nano Banana2
- 生视频平台：Seedance2.0

## 5. 剧本生产分析规则

剧本分析只做生产判断，不写最终 Prompt。必须输出：

- 集数 / 标题 / 场次拆分
- 剧情主线：3-5 句话
- 关键剧情节点：标明不可省略节点
- 人物出入场：谁出现、谁离开、下一组是否仍在场
- 台词与 OS 时长估算
- 动作节点：只作为分镜动作拆解，不作为资产类型
- 初步资产需求：人物 / 场景 / 道具 / 群像 / 特殊状态
- 生产风险与建议分镜策略
- 建议导入的创作身份或专业身份

## 6. 风格锁定规则

人物、场景、道具必须分开锁定，三类风格词不能混用。

人物风格词只管：

- 年龄感
- 身份阶层
- 面部气质
- 体态
- 发型
- 服装结构
- 服装材质
- 妆容
- 首饰
- 表演气质
- 人物连续性锚点
- 人物负面约束

场景风格词只管：

- 建筑类型
- 空间结构
- 入口出口
- 主位与客位
- 可走位区域
- 墙面 / 门窗 / 地面 / 天花材质
- 时代质感
- 色彩系统
- 电影级布光逻辑
- 合乎场景逻辑的实景光源
- 空间负面约束

道具风格词只管：

- 道具名称
- 用途
- 尺寸比例
- 材质
- 纹样
- 结构
- 开合状态
- 可交互部件
- 磨损程度
- 展示角度
- 比例参考
- 细节展示
- 道具负面约束

## 7. 资产拆解规则

资产类型只允许：

- Character
- Scene
- Prop
- Crowd
- SpecialState

禁止把动作关系作为资产类型。掌掴、搀扶、背人、献图、揭面、拔匕、牵袖、对峙、追赶等只进入动作节点、分镜动作拆解或站位参考说明。

资产编号格式：

```text
{集数三位}-{场次两位}-{资产类型英文}-{序号两位}
```

示例：

```text
046-01-Scene-01
046-01-Character-01
046-01-Prop-01
```

资产优先级：

- P0 必需：没有就无法制作关键剧情或关键镜头
- P1 强建议：有助于稳定画面、人物连续性或空间关系
- P2 可选：可替代，但会降低精度
- P3 氛围补充：提升质感，但不是剧情刚需

资产清单必须包含：

- 资产编号
- 资产名称
- 资产类型
- 所属集数 / 场次
- 优先级
- 用途
- 需要视图
- Prompt 平台版本
- 风险备注

## 8. 资产生图 Prompt 规则

资产生图 Prompt 默认交付为 DOCX；如用户只要求文字，可先输出文本版结构。正式资产 Prompt 不使用表格，按资产编号分节，方便复制。

资产 Prompt 可以使用创作身份做前置分析，例如影视美术指导、角色造型设计师、道具设计师、灯光参考设计师。但最终可复制 Prompt 必须写具体设计规格，不写“以某某身份设计”。

人物资产必须包含：

- 场景中景光影图：用于观察人物进入真实场景后的光影、服装材质受光、肤色与场景色温关系。
- 白底三视图 + 近景定妆组合图：用于锁定脸、发型、妆容、首饰、服装领口和整体连续性。

重要场景资产必须包含五视图：

- 顶视图
- 正视图
- 右侧视图
- 左侧视图
- 背视图

道具资产必须包含：

- 比例参考
- 正面 / 侧面 / 背面
- 关键细节特写
- 材质、结构、纹样、可交互部件

平台版本规则：

- 人物资产：GPT Image2 / Nano Banana2
- 场景资产：GPT Image2 / Midjourney / Nano Banana2
- 道具资产：GPT Image2 / Midjourney / Nano Banana2

## 9. 分镜表规则

分镜表是剧本到 Seedance2.0 Prompt 的桥梁，不是文学复述。必须包含：

- 镜头组编号
- 时间码
- 剧情节点
- 场景资产
- 入镜人物
- 核心镜头
- 台词 / OS
- 情绪与表演依据
- 光影设计
- 主镜头组总时长
- 风险
- 覆盖镜头池建议

硬性规则：

- 新场景必须完成空间建立，但建立方式由剧情功能决定，可以是全景、空镜、局部、入场动作、声音先导、遮挡揭示或后续镜头补全；如果它是剧情理解必需镜头，放进主镜头组；如果只是备用剪辑素材，放进覆盖镜头池。
- 每个主镜头只保留一个主动作、一个主空间关系、一个主表演点。
- 每一组必须在本组内建立清楚的人物空间关系，但不强制第一镜完整锚定。第一镜是否锚定，由剧情功能决定；后续镜头只在人物位置、朝向、距离或关键道具状态变化时补充锚定。
- 景别、机位、运镜必须有目的，不能整集重复固定中景。
- 台词和剧情核心不能随意改写。
- 情绪表达必须来自剧本文本、上下文关系、台词语气或前后动作。
- 反打镜头必须写清可见神态或可听语气，不只写“某人看着某人”。
- 光影设计必须绑定具体实景光源，不能只写空泛风格词。
- 每一组必须给出主镜头组总时长；镜头时长精确到 1 秒或 0.5 秒，不使用 0.1 秒。
- 每一组必须判断覆盖镜头池；如果没有覆盖镜头，也要标注“无覆盖镜头”并说明原因。

## 10. 本组镜头策略前置层

正式生成 Seedance2.0 分镜 Prompt 前，必须先为每一组做“本组镜头策略”。这一层只用于导演、摄影、剪辑、声音、表演和 Prompt 工程判断，不进入中文纯 Prompt。

本组镜头策略必须回答：

1. 戏剧目的：本组让观众获得什么剧情信息、情绪信息或动作信息。
2. 信息揭示策略：哪些信息现在必须给，哪些信息可以延迟、遮挡或通过后续镜头揭示。
3. 空间关系建立方式：第一镜锚定、动作中建立、遮挡后揭示、声音先导、第二 / 第三镜补清；必要时判断上一组尾帧姿态是否需要在本组用当前动作或当前结果调整。
4. 表演重点：本组核心表演是台词、沉默、反应、动作停顿、眼神、手部还是身体距离。
5. 运镜策略：固定为主还是安排 1-2 个轻微运镜；长台词镜头是否固定；短动作镜头是否承担运镜。
6. 光影策略：全组光源纪律是什么，是否需要某个关键镜头补一句可见光影结果。
7. 声音策略：开口台词、画外 OS、环境声、静默或口型遮盖风险。
8. 覆盖镜头需求：是否需要空间修复、反应、道具插入、情绪空镜、口型遮盖或动作桥接。
9. 风险判断：资产锚定、站位、时长、纯 Prompt 污染、特效触发词、人物突然出现、场景混用。

推荐输出格式：

```text
【本组镜头策略】
戏剧目的：
信息揭示：
空间关系：
表演重点：
运镜策略：
光影策略：
声音策略：
覆盖镜头需求：
风险：
```

本组镜头策略不追求长篇分析，每项只写能指导 Prompt 的判断。若剧情功能不明确，例如新人物是直接揭示还是延迟揭示，必须先向用户确认。

策略层允许做合理导演调度推理。例如上一组人物半跪、弯腰、背身、伏地、持物、靠近他人，下一组进入较长对话、喊人、转向、离场或上位压迫时，可以在策略层判断是否需要让人物起身、转身、后退、换手持物、调整距离或改变朝向。

这类判断必须遵守两层分工：

```text
策略层 / 检查层：可以写“上一组凤昭半跪拍脸后，D组进入较长对话，为避免长时间半跪造成姿态僵硬，并保持上位压迫，D组开头让凤昭起身或已起身为站姿。”

中文纯 Prompt：只写当前可见动作或当前可见结果，例如“凤昭从半跪姿态起身，站到晏惊弦正前方半步外。”或“凤昭已经站在晏惊弦正前方半步外，身体微微俯向晏惊弦。”
```

纯 Prompt 禁止写：

```text
因为上一组凤昭半跪，所以现在她起身。
为了避免人物长时间半跪造成姿态僵硬，凤昭站起来。
```

## 11. 指令负载与镜头组拆分规则

15 秒镜头组内，纯 Prompt 不追求一次性塞满所有导演信息。每组优先保证：

1. 镜头结构
2. 人物动作
3. 站位关系
4. 台词与口型同步

景深、光影、风格、材质、氛围、音效属于弱控制，应简短、统一、克制。

推荐承载：

- 单镜头长镜头：最稳，可承载 8-12 条核心指令。
- 2-3 个镜头：短剧剧情对话最推荐，每镜 5-8 条核心指令。
- 4 个镜头：可做但要压缩动作，每镜 4-6 条核心指令。
- 5-6 个镜头：高风险，只适合简单覆盖镜或快速反应镜，每镜 3-5 条核心指令。
- 6 个以上镜头：不建议放进同一个 15 秒生成，应拆成下一组。

每镜约束：

- 景别：每镜 1 个
- 运镜：每镜 0-1 个，不要同镜又推又摇又绕
- 主体人物：1-3 人最稳，4 人以上容易串位或弱化
- 人物站位：每镜 1 组主关系
- 主动作：每镜 1 个，复杂动作拆节点
- 表情 / 情绪：每镜 1 个主表演点
- 说话 / 台词：每镜 1 个说话人最稳
- 景深：每镜最多 1 个
- 光影：全组 1-2 条，全局统一；关键镜头只补一句可见结果
- 风格词：全组 3-6 个短风格词，不堆长串
- 音效 / 环境声：全组 1-2 个主声音
- Negative Prompt：全组固定，不逐镜重复

单独生成独立镜头时，可控指令可以增加。可根据剧情需要强化表演节奏、光影、景别和运镜，但仍必须遵守所见即所得和单镜头主动作原则。

低风险运镜策略：

- 每个镜头最多 1 个运镜。
- 每组只给 1-2 个镜头安排轻微运镜。
- 其余镜头优先固定镜头。
- 长台词镜头优先固定，保证口型、表演和台词同步。
- 短动作镜头、入场离场镜头、情绪转折镜头承担轻微运镜。
- 运镜必须服务剧情，例如逼近、离场、发现、压迫、转身、视线转移。
- 不为了电影感而每镜都动。

推荐：

```text
镜头一，固定镜头。
镜头二，轻微推近。
镜头三，固定镜头。
镜头四，轻微跟随。
```

不推荐：

```text
镜头一，推近并横移。
镜头二，环绕并摇镜。
镜头三，跟拍并推近。
```

## 12. Seedance2.0 单文本框输出结构

Seedance2.0 界面只有一个文本框时，可复制 Prompt 按以下顺序组织：

```text
全局风格与光影约束：
……

生成约束 / Negative Prompt：
……

音频约束：
……

角色资产锚定：
……

场景资产锚定：
……

主镜头组：
镜头一……
音频同步……
镜头二……
音频同步……
```

全局区放在镜头正文前面。全局区不是中文纯 Prompt，允许写生成约束和 Negative Prompt。

角色资产名必须保持用户给定的完整 token，不得拆开。比如 `@凤昭墨色暗纹夜间便服` 是一个资产名时，不能改成 `@凤昭，墨色暗纹夜间便服`。

场景资产锚定必须使用 `镜头使用名=@资产登记名`。等号前面是镜头正文中使用的场景名字或描述词，等号后面是 `@` 加资产登记名字。镜头正文第一场景名必须命中等号前名称，不能自由改写成“王府廊下庭院交界”“王府木廊内侧”等未锚定描述。

不正确：

```text
场景资产参考：@王府走廊01正侧视
镜头一，持续3秒。王府廊下庭院交界，竖屏全景。
```

正确：

```text
场景资产锚定：
王府走廊01正侧视=@王府走廊01正侧视

镜头一，持续3秒。王府走廊01正侧视，竖屏全景。
```

推荐全局模板仅作基础结构，正式生成时必须按当前场景资产、时间、空间光源和材质定制，不机械套用默认模板。

```text
全局风格与光影约束：
古风真人写实历史剧质感，夜景低照度，现实主义电影光线，光源只来自画面内烛火、灯笼、窗外冷蓝月色和可见实景光源。人物脸部允许半边落入暗部，暗部不被提亮，保持阴影层次和面部轮廓稳定。真实材质，克制华丽，人物与环境连续一致，连续叙事。

生成约束 / Negative Prompt：
no subtitles, no captions, no on-screen text, no background music, no score, no frontal fill light, no beauty fill light, no studio softbox, no evenly bright face, no HDR brightening.

音频约束：
only character dialogue, lip sync, and natural ambient sound.

角色资产锚定：
凤昭=@凤昭墨色暗纹夜间便服，晏清和=@晏清和，晏惊弦=@晏惊弦，月弦=@月弦侍女服。

场景资产锚定：
别院正视图=@别院正视图，别院背视图=@别院背视图。
```

如项目有特殊平台习惯，可保留英文 Negative Prompt。禁止字幕和禁止音乐是全局生成约束，不写进每个镜头正文。

场景定制示例：

```text
全局风格与光影约束：
古风真人写实历史剧质感，夜内烛光与窗外冷蓝月色统一，现实主义电影光线，光源来自廊下灯笼、室内烛光和庭院冷蓝月色。真实木柱、石板地、竹影和灯笼材质，克制华丽，人物面部稳定，动作节点清楚，人物与环境连续一致，连续叙事。景深按镜头景别控制，近景浅景深，中远景保持空间关系清楚。
```

## 13. 光影控制规则

光影采用“全局纪律 + 关键镜头局部可见结果”的方式。

全局光影负责长期控制：

- 夜景低照度
- 只允许画面内烛火、灯笼、窗外月色、门外冷光等实景光源
- 人物脸部允许半边落入暗部
- 暗部不被自然补光提亮
- 不使用正面美容补光、棚拍柔光箱、均匀亮脸、HDR 式提亮

具体镜头只写可见光影结果，不重复完整规则。

可写：

```text
烛光只照亮凤昭左侧脸，右侧脸落在暗部。
窗外冷蓝月光落在晏清和肩线和后景竹影上。
晏惊弦半张脸被背后竹影压暗，碧眸边缘有冷光。
```

不写：

```text
不要自然补光，不要正面补光，不要均匀亮脸。
```

这些放进全局 Negative Prompt。

## 14. 氛围营造规则

氛围建议两步法：

第一版：干净生产版。只保留资产、站位、主动作、台词同步、基础光源，先保证人物、口型、场景和动作稳定。

第二版：局部氛围加工版。只对关键镜头加 1 个氛围点，不全镜头堆叠。

可用氛围点：

- 夜风吹动鬓边碎发
- 后景烛火轻轻跳动
- 竹叶影子在白衣上缓慢晃过
- 窗纸上映出细碎竹影
- 地面月光被门框切成一条冷色光带
- 衣袖边缘被风轻轻带动

每个镜头最多一个主氛围点。台词镜头优先保留嘴部动作、口型同步和音频同步，氛围从简。

## 15. 语言情绪与表演节奏规则

语言情绪是声音层指令，写在音频同步里。表演节奏是画面层指令，写在镜头正文里。

语言情绪可以写，但必须绑定可听声音状态。抽象情绪词只能作为辅助，不得单独作为控制指令。

推荐写法：

```text
音频同步：晏惊弦使用参考音频51-1-E1，声音短促，音量压低，尾音绷紧，说出“你！”
```

可接受但需要克制：

```text
音频同步：晏惊弦使用参考音频51-1-E1，以短促、压着怒意、被气住的语气说出“你！”
```

更稳的方式是把抽象情绪拆成声音和表演：

```text
晏惊弦肩背微微僵住，嘴唇短促张开，开口说话，口型跟随参考音频。
音频同步：晏惊弦使用参考音频51-1-E1，声音短促，音量压低，尾音绷紧，说出“你！”
```

内心 OS 必须和开口台词分开写。OS 镜头中，画面层必须明确角色不张嘴，不能写“开口说话”“口型跟随参考音频”。音频同步层必须明确这是画外内心声，尽量使用“响起”而不是“说出”，降低人物张嘴风险。

推荐 OS 写法：

```text
凤昭不张嘴，嘴唇保持闭合，视线停住，呼吸很轻。
音频同步：凤昭使用参考音频51-1-C2，OS为画外内心声，近距离声音很轻，响起：“十年？”
```

如果需要更强控制，可写：

```text
凤昭不张嘴，嘴唇保持闭合，脸部没有说话口型，视线停在画面右侧，呼吸很轻。
音频同步：凤昭使用参考音频51-1-C2，OS为画外内心声，近距离、很轻、贴近耳边的内心声响起：“十年？”
```

OS 禁止写：

```text
凤昭嘴唇张开，开口说话，口型跟随参考音频。
音频同步：凤昭使用参考音频51-1-C2，说出“十年？”
```

画面层可写：

- 唇线收紧
- 停顿半秒后开口
- 视线停在某处
- 肩背微微僵住
- 手指收紧
- 呼吸变浅
- 身体向前压近半步

避免写容易被模型当成突发视觉特效的词。此类词不是完全不能出现在检查层，但不能直接写进中文纯 Prompt，除非项目明确就是奇幻、法术、超自然、特效风格，且用户确认需要这种效果。

高风险词类型：

1. 面色突变类：脸色惨白、脸色刷白、面色煞白、血色褪尽、突然变白、面如死灰。
2. 发光发亮类：眼睛发光、瞳孔亮起、身上发光、周身泛光、金光闪过、冷光从体内透出。
3. 气场外放类：杀气四溢、气场爆开、威压扩散、压迫感弥漫、寒意蔓延、黑气缠绕。
4. 粒子烟雾类：黑雾升起、白雾缠绕、烟气从身上冒出、空气扭曲、尘埃炸开。
5. 突然变形类：表情瞬间崩裂、眼神瞬间变红、面部扭曲、身体僵成雕像。
6. 抽象情绪视觉化类：恐惧笼罩全身、心碎感扩散、怒意燃烧、情绪爆发成一片冷光。
7. 镜头特效类：画面炸裂、画面震碎、时间静止、世界凝固、空气凝结、空间扭曲。
8. 呼吸白气类：轻吐一口气、吐出一口气、呼出一口气、哈出一口气、白气从口中散开。

这些词容易生成脸部变色、发光、粒子、烟雾、滤镜、震屏、超自然或奇幻特效。

如果要表达震惊、恐惧、失血感或情绪受击，优先改成可见表演、动作停顿、光影结果或已建立状态：

```text
凤昭呼吸停住，唇线绷紧，手指在袖口边缘收紧。
凤昭眼神停在画面右侧，肩背微微僵住。
冷蓝月光落在凤昭侧脸，脸部暗部更重。
凤昭站在灯笼光外，半张脸落在阴影里。
```

替代表达原则：

- “杀气四溢”改为：眼神压低、身体向前压近半步、声音压低。
- “压迫感弥漫”改为：低位镜头，角色站在前景，身影压住后景人物。
- “眼睛发光 / 眼神变红”改为：眼神停住、眼眶边缘泛湿、瞳孔反射烛光。
- “黑气缠绕 / 寒意蔓延”改为：后景冷蓝月光更重，角色半张脸落在暗部。
- “画面震碎 / 世界凝固”改为：固定镜头，人物停顿，环境声短暂变轻。
- “怒意燃烧”改为：唇线压紧，手指收紧，语速变慢，尾音绷住。
- “轻吐一口气 / 吐出一口气”改为：肩背慢慢放松，唇线松开，呼吸放缓。

呼吸白气只有在明确冬天、寒夜、雪地、严寒室外等环境中才可写，并且必须写成可见环境结果：

```text
冬夜庭院里，凤昭呼出的白气在唇边短暂散开。
```

如果不是寒冷环境，不写“吐气”“白气”“哈气”，只写放松后的身体状态。

如果“苍白 / 病弱 / 失血”是人物长期状态或特殊状态资产，必须在人物资产、妆造设定或 SpecialState 资产中提前锁定，不在单个镜头纯 Prompt 中临时写成突发变化。

避免在纯 Prompt 中只写：

- 心中起疑
- 暗中试探
- 已经识破
- 杀意已起
- 制造压迫感

这些只能进入检查层。

## 16. 覆盖镜头池规则

覆盖镜头池是备用剪辑素材，用于剪辑补位、情绪转场、动作强调、空间修复、节奏缓冲和口型风险规避。内部计算时，覆盖镜头不并入主镜头组总时长，但每条覆盖镜头本身仍是要生成的视频镜头，必须有自己的持续时间。

覆盖镜头池必须按每一组镜头判断和书写，不等整集做完后才首次设计。整集完成后可以追加一次“全片覆盖镜头补漏”，用于检查是否缺少统一空镜、关键道具插入、转场或口型遮盖素材。

覆盖镜头池正式交付时必须单列，但输出内容里不要写“独立生成”“不计入主时长”“供后期剪辑使用”等说明性文字。

覆盖镜头池不能压缩成概念句。只要写“有覆盖镜头”，每条覆盖镜头都必须满足可直接生成结构：

1. 覆盖镜头编号。
2. 持续时间，精确到 1 秒或 0.5 秒。
3. 场景资产使用名，必须命中场景资产锚定等号前名称。
4. 景别。
5. 机位 / 运镜，每条最多 1 个运镜。
6. 可见主体；空镜必须写清可见环境主体。
7. 唯一主动作；空镜必须写清可见环境变化。
8. 声音状态：无台词 / 环境声 / 可听动作声 / OS。

```text
主镜头组：
……

覆盖镜头池：
覆盖镜头一，持续2秒。场景镜头使用名，中景，固定机位，50mm，中等景深。角色A站在门框内侧，视线停在画外角色B离开的方向，袖口垂在身侧。廊下灯笼轻微晃动，只有很轻的夜风声。
覆盖镜头二，持续2.5秒。场景镜头使用名，特写，固定机位，85mm，浅景深。桌案边缘的烛火跳动一下，窗纸上的竹影缓慢偏移，房间里没有人物入镜。环境声很轻。
```

每组检查层必须写：

```text
覆盖镜头池：有 / 无
覆盖镜头用途：空间修复 / 反应 / 道具插入 / 情绪空镜 / 口型遮盖 / 动作桥接 / 新场景建立 / 入场离场衔接
```

如果没有覆盖镜头，也必须标注：

```text
覆盖镜头池：无。本组主镜头已完成空间建立、台词同步和动作闭合，暂不需要覆盖镜头。
```

覆盖镜头类型：

1. 空间修复镜头：交代人物在哪里、谁和谁的位置变化。
2. 动作桥接镜头：连接动作节点，避免剪辑跳跃。
3. 反应镜头：补听者或旁观者的可见反应。
4. 道具插入镜头：强调剧情信息，如匕首、卷轴、茶盏、血迹、袖口。
5. 情绪转场空镜：用烛火、竹影、空椅、窗纸、门框等承接情绪。
6. 视线承接镜头：交代角色在看什么。
7. 权力关系镜头：强化压迫、距离、主从、对峙。
8. 遮挡 / 隐藏信息镜头：让动作发生但不完全暴露。
9. 节奏缓冲镜头：给台词密集或动作太快的段落留呼吸。
10. 口型风险规避镜头：用听者反应、手部动作、道具特写、背影或环境空镜遮盖口型不稳。
11. 新场景建立镜头：如果是剧情理解必需，放主镜头；如果是备用剪辑，放覆盖镜头池。
12. 离场 / 入场衔接镜头：补人物进出场和方向关系。

万能空镜必须有具体意义，不能只写“给一个空镜”。例如：

```text
覆盖镜头一，持续2秒。场景镜头使用名，特写，固定机位，85mm，浅景深。桌案边缘的烛火跳动一下，窗纸上的竹影缓慢偏移，房间里没有人物入镜。环境声很轻。
```

## 17. Seedance2.0 主镜头写作规则

每个镜头写纯 Prompt 前，必须先判断：

1. 当前镜头使用的场景背景资产。
2. 当前镜头可见人物。
3. 当前镜头可见道具。
4. 当前镜头的主空间关系。
5. 当前镜头唯一主动作。
6. 当前镜头是开口台词、画外 OS，还是无声音。
7. 当前镜头是否需要一个局部光影结果。
8. 当前镜头是否需要一个氛围点。
9. 删除所有解释性、推理性、规则性、资产说明性语言。
10. 只保留观众能看见和听见的内容。

视频生成模型不能被假设会可靠继承上一组的空间、站位、人物朝向、距离关系、动作尾帧或手持道具状态，所以每一组必须在本组内建立观众能理解的人物空间关系。但站位锚定不一定要在第一镜完成，第一镜是否锚定必须服从剧情功能。

如果第一镜是对话、对峙、多人物同场、近距离动作、递物、拉扯、持刀、人物站位影响台词理解，第一镜应做轻量站位锚定。

如果第一镜是新场景建立、空镜、入场、走动、追赶、闪避、突发动作、新人物首次出现、悬念揭示、遮挡构图、先声后人，可以通过动作、遮挡、局部、声音或后续镜头建立关系，不强制第一镜完整锚定。

新人物首次出现时，不固定使用遮挡、背影、局部、脚步声或直接正脸；必须根据剧情功能判断。若不明确，应向用户确认是“直接揭示”还是“延迟揭示”。

轻量站位锚定至少包含：

- 场景镜头使用名
- 主要人物位置：左前景 / 右中景 / 后景 / 门口 / 桌案前等
- 人物朝向和相对关系
- 人物距离：半步 / 一步 / 隔着桌案 / 隔着门槛等
- 关键道具或状态：手持卷轴、站在木柱旁、袖口垂下等

可写：

```text
凤昭站在画面左前景，身体朝向画面右侧的晏清和。晏清和站在画面右中景，身体朝向凤昭，两人相距半步。月弦站在凤昭身后一步外，半身被木柱遮住。
```

连续动作组或延后锚定时，只能写当前可见结果，不写上一组路径解释。

跨组姿态调整时，可以写当前可见动作或当前可见结果。允许写“凤昭从半跪姿态起身”“凤昭已经站在晏惊弦正前方半步外”，但禁止写“因为上一组她半跪所以现在站起来”。

可写：

```text
凤昭已经站在廊下木柱旁，身体朝向庭院方向。
凤昭从半跪姿态起身，衣摆从石板地上垂落下来。
凤昭已经站在晏惊弦正前方半步外，身体微微俯向晏惊弦。
```

不写：

```text
凤昭从上一组走到廊下木柱旁。
因为上一组凤昭半跪，所以现在凤昭起身。
```

推荐镜头句式：

```text
镜头一，持续3秒。场景镜头使用名，景别，机位 / 运镜，焦段，景深。角色A站位，角色B站位，主动作，可见表演，可见光影或氛围。
音频同步：角色A使用参考音频编号，声音状态，说出“台词”。
```

镜头正文中的场景镜头使用名必须来自“场景资产锚定”等号前名称。角色、道具可以使用用户登记的完整资产 token；场景不要在镜头正文里临时自由改名。

无台词镜头不写音频同步。开口台词镜头必须写“开口说话，口型跟随参考音频”或等价描述。内心 OS 镜头必须写“不张嘴 / 嘴唇保持闭合”，并在音频同步中写“OS为画外内心声”。

每一组检查层必须给出主镜头组总时长和镜头拆分时长：

```text
主镜头组总时长：12秒
镜头时长：镜头一3秒，镜头二3秒，镜头三2.5秒，镜头四3.5秒
覆盖镜头池：有 / 无
```

镜头时长精确到 1 秒或 0.5 秒，不使用 0.1 秒。

## 18. 证据式二次规则检查

正式 Seedance2.0 分镜 Prompt 必须执行生成后三步流程：

1. 生成分镜 Prompt。
2. 执行 V7.1 证据式规则检查。
3. 若有问题，直接输出修正版；若无问题，标注“V7.1 检查通过”。

检查方式：

- 草稿版：可以生成后只做简查，重点看场景资产锚定、纯 Prompt 污染、时长和覆盖镜头池。
- 正式复制版：必须做完整 V7.1 检查并修正后再交付。
- 用户明确说“先别检查，只出草稿”：可以跳过二次检查，但必须标注“未执行 V7.1 二次检查”。

二次检查重点：

- 场景资产锚定是否完整。
- 每个镜头第一场景名是否命中场景资产锚定等号前名称。
- 是否出现未锚定场景描述。
- 纯 Prompt 是否出现解释、推理、规则提醒、资产用途说明或污染词。
- 策略层 / 检查层的导演判断是否没有误写进纯 Prompt。
- 跨组姿态调整是否在纯 Prompt 中只写当前可见动作或当前可见结果，没有写原因解释。
- 是否写入不可见心理、动机、身份真相、剧情因果。
- 每组是否有主镜头组总时长。
- 单镜头时长是否只到 1 秒或 0.5 秒。
- 覆盖镜头池是否与主镜头组分开书写。
- 每组是否判断覆盖镜头池“有 / 无”。
- 覆盖镜头不并入主镜头组总时长时，是否仍然给每条覆盖镜头写明自己的持续时间。
- 每条覆盖镜头是否满足可直接生成结构：编号、持续时间、场景资产使用名、景别、机位 / 运镜、可见主体或空镜主体、唯一主动作或环境变化、声音状态。
- 每条覆盖镜头的场景资产使用名是否命中场景资产锚定等号前名称。
- 覆盖镜头是否被压缩成概念句、用途说明句或剪辑说明句；如是，必须改写为完整镜头 Prompt。
- 每组是否在本组内建立了观众能理解的人物空间关系。
- 第一镜未完整锚定时，是否由剧情功能支持，并在后续镜头补清关系。
- 站位锚定或延后锚定是否只写当前可见事实，没有写“从上一组走到”等画面外路径解释。
- 开口台词镜头是否有音频同步和口型跟随参考音频。
- 内心 OS 镜头是否明确角色不张嘴、嘴唇保持闭合，且音频同步写为画外内心声。
- 禁止字幕和音乐是否只在全局生成约束中出现。
- 是否出现容易触发突发视觉特效的词：面色突变、发光发亮、气场外放、粒子烟雾、突然变形、抽象情绪视觉化、镜头特效类表达。
- 是否每个镜头最多 1 个运镜，且每组只有 1-2 个镜头使用轻微运镜。
- 长台词镜头是否优先固定，短动作镜头是否承担运镜。
- 15 秒内镜头数量、动作、站位、表演、光影、氛围是否过载。

二次检查必须采用证据式输出。每个问题必须包含检查项、结果、证据、修正方式。不能只写“已检查”“通过”“无问题”。

二次检查输出格式：

```text
【V7.1 证据式二次规则检查】
结论：通过 / 需要修正

问题：
1. 检查项：
   结果：通过 / 不通过
   证据：引用原句或指出缺失位置
   修正：给出修正后的句子或结构

处理：
已输出修正版 / V7.1 检查通过 / 未执行 V7.1 二次检查
```

覆盖镜头必须逐条检查，不能只检查覆盖镜头池是否存在。示例：

```text
检查项：覆盖镜头一是否可直接生成
结果：不通过
证据：原文只有“情绪空镜：烛火跳动”，缺少持续时间、场景资产使用名、景别、机位和声音状态。
修正：覆盖镜头一，持续2秒。场景镜头使用名，特写，固定机位，85mm，浅景深。桌案边缘的烛火跳动一下，窗纸上的竹影缓慢偏移，房间里没有人物入镜。环境声很轻。
```

示例：

```text
检查项：场景资产锚定
结果：不通过
证据：镜头一写“王府廊下庭院交界”，场景资产锚定表没有该名称。
修正：改为“王府走廊01正侧视”。
```

## 19. 终检清单

每个阶段交付前必须终检。

通用终检：

- 是否按阶段推进？
- 是否区分生图平台和生视频平台？
- 人物 / 场景 / 道具风格锁定词是否分离？
- 资产类型是否只包含 Character / Scene / Prop / Crowd / SpecialState？
- 动作关系是否没有被列为资产类型？
- 资产编号是否符合格式？
- 资产 Prompt 是否不使用表格？
- 场景五视图、人物两类图、道具比例参考是否完整？

分镜终检：

- 新场景是否完成空间建立，建立方式是否服务剧情？
- 每个主镜头是否只有一个主动作和一个主空间关系？
- 每组是否在本组内建立清楚的人物空间关系？
- 第一镜未完整锚定时，是否由新场景建立、入场、走动、突发动作、新人物首次出现、悬念揭示、遮挡构图或先声后人等剧情功能支持？
- 站位锚定或延后锚定是否使用当前可见事实，而不是上一组路径解释？
- 景别、机位、运镜是否服务剧情？
- 是否执行低风险运镜策略：每镜最多 1 个运镜，每组 1-2 个轻微运镜，其余固定？
- 长台词镜头是否优先固定？
- 情绪表达是否有剧本依据？
- 覆盖镜头池是否单列，且没有写入说明性文字？
- 每一组是否给出主镜头组总时长？
- 镜头时长是否只精确到 1 秒或 0.5 秒？
- 每一组是否判断覆盖镜头池“有 / 无”，无覆盖镜头时是否说明原因？
- 有覆盖镜头时，每条覆盖镜头是否都有自己的持续时间，且不被误省略？
- 每条覆盖镜头是否都具备可直接生成结构：编号、持续时间、场景资产使用名、景别、机位 / 运镜、主体、主动作、声音状态？
- 覆盖镜头是否没有被写成“空间修复镜头：……”或“情绪空镜：……”这类不可直接生成的概念句？

Seedance2.0 终检：

- 全局约束是否放在镜头正文前面？
- 生成约束是否包含 no subtitles / no captions / no on-screen text / no background music / no score？
- 角色资产名是否保持完整 token，没有被拆开？
- 场景资产锚定是否使用 `镜头使用名=@资产登记名`？
- 镜头正文第一场景名是否命中场景资产锚定等号前名称？
- 是否避免把未锚定的场景描述当作镜头第一场景名？
- 纯 Prompt 是否只写当前镜头可见、可听内容？
- 是否把检查层判断写进了纯 Prompt？
- 是否写了观众看不见的心理、动机、身份真相、剧情因果？
- 是否出现“为了表现、用于表现、暗示、说明、代表、意味着、基于参考图、不要、避免、可以、或者、必要时”等污染词？
- 是否出现容易触发突发视觉特效的词：面色突变、发光发亮、气场外放、粒子烟雾、突然变形、抽象情绪视觉化、镜头特效类表达？
- 是否在远景、背影、遮挡镜头中写了不可见表情或手部细节？
- 是否因为想让模型理解剧情而堆叠过多解释？
- 是否存在人物突然出现、场景背景混用、门窗角色混乱、方向词无参照物的问题？
- 是否符合“所见即所得”第一原则？

只要 Seedance2.0 终检任一项不合格，必须回到检查层重写，不得直接交付纯 Prompt。

## 20. Seedance 实测失败反馈记录

每次 Seedance2.0 实测后，如出现明显失败，必须记录失败类型，并判断是否需要沉淀为规则补丁。该记录用于让规则从真实生成结果中进化，不进入中文纯 Prompt。

失败记录格式：

```text
【Seedance 实测失败记录】
项目 / 集数 / 镜头组：
失败类型：资产调用 / 站位 / 口型 / OS 张嘴 / 字幕 / 音乐 / 光影 / 运镜 / 特效误触发 / 人物突然出现 / 场景混用 / 时长 / 其他
问题镜头：
原 Prompt：
失败现象：
原因判断：
修正版：
是否沉淀为规则补丁：是 / 否
补丁内容：
```

高频失败必须进入规则补丁。单次偶发失败可以只记录，不立即改规则。

```text
只返工第 X 场分镜，不动资产清单。
```

