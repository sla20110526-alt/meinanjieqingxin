# 短剧 AI 视频生产使用框架

> 历史版本提示：本文件已被《短剧AI视频生产规则_V7_统一版.md》吸收。后续新项目以 V7 为唯一活动规则源，本文件仅作历史参考。

用途：指导你后续如何正确使用我来做短剧规划、剧本分析、资产拆解、资产生图 Prompt、分镜表和 Seedance2.0 分镜 Prompt。

配套规则：`短剧AI视频生产执行规则.md`

最高优先级补丁：`V6修正规则_纯Prompt第一原则.md`

## 1. 总原则

你不要把我当成“一次性写 Prompt 工具”，而要把我当成“短剧 AI 视频生产统筹”来用。

正确用法是：

1. 先定项目规则
2. 再分析剧本
3. 再拆资产
4. 再做资产图 Prompt
5. 再做分镜表
6. 最后做 Seedance2.0 分镜 Prompt

如果一开始就让我直接写最终视频 Prompt，容易出现资产缺失、人物突然出现、场景串用、动作关系误当资产、镜头不可执行、纯 Prompt 被解释性文字污染等问题。

## 2. 你后续可以用三种模式调用我

### 模式 A：新项目从零启动

适合你要做一部新短剧、一个新系列、一个新账号方向。

你可以这样发：

```text
我要启动一个新短剧 AI 视频项目。
请先帮我建立项目控制面板，不要直接写分镜 Prompt。

项目类型：
剧集规模：
单集目标时长：
画幅：
生图平台：
生视频平台：
已有资料：
我本次想先做到：
```

我应该输出：

- 项目控制面板
- 需要你补充的资料清单
- 本项目执行顺序
- 当前风险
- 下一步建议

### 模式 B：单集生产流程

适合你已经有某一集剧本，要从剧本推进到资产和分镜。

你可以这样发：

```text
这是第 X 集剧本。
请按执行规则先做剧本生产分析，不要写最终 Prompt。

项目背景：
本集目标时长：
画幅：
已有角色 / 场景 / 道具资产：
剧本正文：
```

我应该先输出：

- 场次拆分
- 剧情主线
- 关键剧情节点
- 人物出入场
- 台词与 OS 时长估算
- 动作节点
- 初步资产需求
- 生产风险
- 建议下一步：风格锁定或资产拆解

### 模式 C：局部补做或返工

适合你已经做到一半，只想让我补某个环节。

你可以这样发：

```text
我现在只要补做【资产清单 / 资产生图 Prompt / 分镜表 / Seedance2.0 Prompt】。
已有前置信息如下：

项目规则：
剧本片段：
已确认资产：
已有分镜：
需要返工的问题：
本次只输出：
```

我应该输出你指定的环节，同时标注：

- 哪些前置信息不足
- 哪些地方可能影响后续生产
- 是否建议补做前一阶段

## 3. 标准推进顺序

### 第一步：项目控制面板

你需要给我：

- 项目名称
- 项目类型
- 剧集规模
- 单集目标时长
- 画幅
- 生图平台
- 生视频平台
- 已有参考图或设定
- 本次要做整剧、单集，还是单个环节

我输出后，你确认：

```text
确认项目控制面板，进入剧本分析。
```

### 第二步：剧本分析

你需要给我：

- 单集剧本
- 集数编号
- 是否有固定标题
- 目标时长
- 是否必须保留全部台词

我输出：

- 剧情主线
- 关键节点
- 人物出入场
- 台词 / OS 时长估算
- 动作节点
- 初步资产需求
- 风险

你确认：

```text
确认剧本分析，进入风格锁定。
```

或：

```text
剧本分析里第 X 点需要修改：……
```

### 第三步：风格锁定

你需要给我：

- 项目题材
- 视觉风格偏向
- 人物设定
- 场景设定
- 道具设定
- 参考图说明，如果有

我输出：

- 人物风格锁定词
- 场景风格锁定词
- 道具风格锁定词
- 各自负面约束
- 平台版本

你确认：

```text
确认风格锁定，进入资产拆解。
```

### 第四步：资产拆解

你需要给我：

- 已确认剧本分析
- 已确认风格锁定
- 是否已有资产图
- 是否需要按集数和场次编号

我输出：

- Character 资产
- Scene 资产
- Prop 资产
- Crowd 资产
- SpecialState 资产
- P0 / P1 / P2 / P3 优先级
- 缺图风险
- 动作节点与站位参考说明

你确认：

```text
确认资产清单，进入资产生图 Prompt。
```

### 第五步：资产生图 Prompt

你需要给我：

- 已确认资产清单
- 每类资产对应平台
- 是否要输出 DOCX
- 是否只先做 P0 / P1 资产

我输出：

- 按资产编号分节的资产 Prompt
- 人物：场景中景光影图、白底三视图 + 近景定妆
- 场景：五视图 Prompt
- 道具：比例参考与细节展示 Prompt
- 负面约束

你确认：

```text
确认资产 Prompt，进入分镜表。
```

### 第六步：分镜表

你需要给我：

- 已确认剧本分析
- 已确认资产清单
- 单集目标时长
- 是否保留全部台词
- 是否有必须出现的镜头

我输出：

- 镜头组编号
- 时间码
- 剧情节点
- 场景资产
- 入镜人物
- 核心镜头
- 覆盖镜头
- 台词 / OS
- 情绪表达
- 光影设计
- 风险

你确认：

```text
确认分镜表，进入 Seedance2.0 分镜 Prompt。
```

### 第七步：Seedance2.0 分镜 Prompt

你需要给我：

- 已确认分镜表
- 已确认资产清单
- 角色资产名称
- 场景资产名称
- 道具资产名称
- 音频或配音编号，如果有

我输出：

- 检查层
- 用途型资产指向
- 中文纯 Prompt
- 音频同步
- 覆盖镜头
- 终检风险

这一阶段必须执行 V6 纯 Prompt 第一原则：

- 检查层可以写解释、判断、资产用途、风险控制。
- 中文纯 Prompt 只能写当前镜头可见、可听内容。
- 纯 Prompt 不能写剧情解释、导演意图、人物心理判断、资产用途说明、规则提醒、二选一表达。
- 每个镜头优先控制镜头结构、人物动作、站位关系。
- 如果 15 秒镜头组信息过载，应拆组或删减，不用解释性语言强行补足。

你最后确认：

```text
确认本集 Seedance2.0 Prompt，做终检。
```

## 4. 最推荐的对话节奏

最稳的节奏是“一步一确认”：

```text
第一轮：建立项目控制面板
第二轮：剧本分析
第三轮：风格锁定
第四轮：资产拆解
第五轮：资产生图 Prompt
第六轮：分镜表
第七轮：Seedance2.0 Prompt
第八轮：终检与返工
```

每一步你只需要确认三件事：

- 这一步有没有漏掉剧情或资产
- 人物、场景、道具有没有混用
- 下一步是否可以继续

## 6. 你该避免的用法

尽量不要这样用：

```text
帮我把这个剧本直接写成 Seedance Prompt。
```

更好的写法是：

```text
请先按工作流做剧本生产分析，不要直接写最终 Prompt。
分析完成后等我确认，再进入资产拆解。
```

也不要这样用：

```text
帮我把所有人物、场景、动作都列成资产。
```

更好的写法是：

```text
请按 Character / Scene / Prop / Crowd / SpecialState 拆解资产。
动作关系不要作为资产类型，只写入动作节点或站位参考说明。
```

## 7. 快速指令模板

### 只做剧本分析

```text
请按执行规则分析以下短剧剧本，只做生产分析，不写最终 Prompt。
输出：场次拆分、剧情主线、关键节点、人物出入场、台词时长、动作节点、初步资产需求、生产风险。
```

### 只做资产清单

```text
请根据已确认的剧本分析拆解资产清单。
资产类型只允许 Character / Scene / Prop / Crowd / SpecialState。
动作关系不要作为资产类型。
资产编号使用 046-01-Scene-01 这种格式。
```

### 只做资产生图 Prompt

```text
请根据资产清单制作资产生图 Prompt。
按资产编号分节，不使用表格。
人物输出场景中景光影图、白底三视图 + 近景定妆。
场景输出五视图。
道具输出比例参考与细节展示。
```

### 只做分镜表

```text
请根据剧本分析和资产清单制作分镜表。
新场景必须有全景或空镜建立。
每组镜头必须有 1-3 个覆盖镜头。
景别、机位、运镜要有目的。
光影必须绑定合理实景光源。
```

### 只做 Seedance2.0 Prompt

```text
请根据已确认分镜表和资产清单制作 Seedance2.0 分镜 Prompt。
输出结构：检查层 + 用途型资产指向 + 中文纯 Prompt。
纯 Prompt 只写所见即所得，不写规则解释、资产解释、不可见心理和模糊指代。
```

## 8. 返工方式

如果你不满意，不要只说“重写”。建议具体指出返工范围：

```text
只返工资产清单，不动剧本分析。
```

```text
只修改第 2 场分镜，保持原台词不变。
```

```text
Seedance Prompt 里纯 Prompt 有解释性文字，请只清理纯 Prompt，不改检查层。
```

```text
人物情绪加得太满，请根据剧本文本收敛微表情。
```

这样我能精确修改，不会把已经确认的内容推翻。

## 9. 最小启动流程

如果你现在还没有完整规划，最简单的启动方式是直接发：

```text
我要开始一个短剧 AI 视频项目。
请按工作流先帮我建立项目控制面板，并告诉我还需要补充哪些资料。
不要直接写分镜 Prompt。
```

如果你已经有单集剧本，直接发：

```text
这是第 X 集剧本。
请按工作流先做剧本生产分析，不要直接写资产 Prompt 或 Seedance Prompt。
分析完成后等我确认。
```

## 10. 默认协作约定

后续你没有特别说明时，我默认按以下方式处理：

- 先分析，再生产
- 先资产，再分镜
- 先分镜表，再 Seedance2.0 Prompt
- 动作关系不列为资产
- 人物、场景、道具风格词分离
- 资产 Prompt 不使用表格
- 纯 Prompt 只写所见即所得
- Seedance2.0 纯 Prompt 必须执行 V6：检查层负责解释，纯 Prompt 只保留当前镜头可见画面和可听声音
- 每个关键阶段完成后等待你确认再进入下一阶段
