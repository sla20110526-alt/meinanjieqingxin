# V6 修正规则｜纯 Prompt 第一原则：所见即所得

> 历史版本提示：本文件已被《短剧AI视频生产规则_V7_统一版.md》吸收。后续新项目以 V7 为唯一活动规则源，本文件仅作历史参考。

本规则为 Seedance2.0 分镜 Prompt 阶段的最高优先级修正规则。凡涉及“中文纯 Prompt”的输出，必须优先执行本规则。

## 一、最高优先级原则

纯 Prompt 只描写当前镜头中观众能够直接看见、直接听见的内容。

所有纯 Prompt 必须遵守“所见即所得”原则：画面中能呈现什么，就写什么；声音中能听见什么，就写什么。不能为了让模型理解剧情，把导演解释、剧情意图、人物心理判断、资产用途说明、空间推理过程、隐藏因果、不可见动作路径、规则提醒、生成建议写进纯 Prompt。

该规则优先级高于美术风格、镜头丰富度、运镜设计、动作完整度、台词节奏、情绪解释、资产调用说明和导演意图表达。

## 二、纯 Prompt 允许写入的内容

纯 Prompt 允许写入以下可见、可听内容：

1. 当前镜头的场景背景

   例如：凤昭书房外客厅左侧视图、窗棂、夜色、灯笼、座椅、地毯、门框。

2. 当前镜头可见人物

   例如：凤昭站在桌案前，凤凛川站在凤昭正前方，两人相距半步。

3. 当前镜头可见动作

   例如：凤昭抬手按住凤凛川手腕，凤凛川身体向前压近，匕首刀柄贴近凤昭袖口。

4. 当前镜头可见表演

   例如：凤昭唇线收紧，视线停在凤凛川手中匕首上，肩背微微绷住。

5. 当前镜头可见空间关系

   例如：凤昭在画面左前景朝右，凤凛川在画面右中景朝左，二人之间隔着半步地毯。

6. 当前镜头可见镜头语言

   例如：中近景，轻微推近，低位跟拍，固定镜头，浅景深。

7. 当前镜头可听见的声音

   例如：凤昭开口说话，口型跟随参考音频；凤昭不张嘴，OS 为画外内心声；匕首划过衣袖发出短促摩擦声。

8. 当前镜头可呈现的光影与质感

   例如：夜内烛光从画面左侧照亮凤昭侧脸，窗外冷蓝月色落在后景窗棂上。

## 三、纯 Prompt 禁止写入的内容

纯 Prompt 禁止写入以下内容：

1. 剧情解释

   禁止写：“为了表现凤昭已经识破凤凛川的身份。”

   应改为：“凤昭视线停在凤凛川袖口，手指缓慢收紧。”

2. 人物心理判断

   禁止写：“凤昭心中已经起疑。”

   应改为：“凤昭笑意停住，目光从卷轴移到凤凛川手腕。”

3. 导演意图

   禁止写：“这一镜用于制造压迫感。”

   应改为：“镜头低位轻微推近，凤凛川的身影压近凤昭肩前。”

4. 资产用途说明

   禁止写：“使用图3作为站位参考，不作为背景。”

   应放入检查层，不进入纯 Prompt。

5. 画面外动作路径

   禁止写：“凤昭从上一组的位置走到这里。”

   应改为当前可见结果：“凤昭站在桌案前沿，身体朝向凤凛川。”

6. 规则提醒

   禁止写：“不要混淆人物，不要生成多余角色。”

   应放入 Negative Prompt 或检查层，不进入纯 Prompt。

7. 二选一或不确定表达

   禁止写：“凤凛川可以站在门边或窗边。”

   必须改成确定画面：“凤凛川站在窗棂前方，身体朝向凤昭。”

8. 模型理解辅助语

   禁止写：“基于16:9参考图的画面左侧区域。”

   应改为可见结果：“画面左前景是桌案边沿，凤昭站在桌案右侧。”

9. 不可见细节

   禁止在背影、远景、遮挡镜头中写清晰正脸表情、眼神细节、手指微动作。

   如果需要表情，就必须切换到正面、侧面或近景。

## 四、检查层与纯 Prompt 的分工

检查层负责解释、判断、规划、风险控制。

纯 Prompt 负责呈现当前镜头的可见画面和可听声音。

检查层可以写：

- 当前镜头使用哪张场景图作为可见背景。
- 哪张图只作为动作参考、站位参考、入口参考或反打参考。
- 人物从上一组尾帧如何承接。
- 当前镜头的门、窗、桌案、座椅、帘幔分别承担什么空间功能。
- 哪些信息不能进入纯 Prompt。
- 是否存在提示词污染风险。
- 是否存在人物突然出现、场景混用、方向混乱、动作不可见的问题。

纯 Prompt 不能写这些解释，只写最终画面事实。

## 五、纯 Prompt 写作顺序

每个镜头写纯 Prompt 前，必须先完成以下判断：

1. 锁定当前镜头使用的场景背景资产。
2. 锁定当前镜头可见人物。
3. 锁定当前镜头可见道具。
4. 锁定当前镜头的主空间关系。
5. 锁定当前镜头唯一主动作。
6. 锁定当前镜头是否有人开口说话。
7. 删除所有解释性、推理性、规则性、资产说明性语言。
8. 只保留观众能看见和听见的内容。

## 六、15 秒镜头组的指令减载原则

15 秒镜头组内，纯 Prompt 不追求一次性塞满所有导演信息。每组优先保证三类强控制：

1. 镜头结构
2. 人物动作
3. 站位关系

景深、光影、风格、材质、氛围、音效属于弱控制，应简短、统一、克制。

如果同一组里同时出现多镜头、多人物、复杂站位、复杂动作、长台词、复杂运镜、复杂光影、多道具、多音效，应优先拆组或删减指令，不得用解释性语言强行补足。

## 七、错误示例与修正示例

错误写法：

```text
镜头一，持续4秒。凤昭为了试探凤凛川，假装没有发现他袖中藏刀，站在桌案前制造压迫感。凤凛川心中杀意已起，准备趁献图时刺杀凤昭。镜头表现二人暗中交锋。
```

修正写法：

```text
镜头一，持续4秒。凤昭书房外客厅左侧视图，中近景，固定镜头。凤昭站在桌案前沿，身体朝向凤凛川，唇角带笑，视线停在凤凛川递来的卷轴上。凤凛川站在凤昭正前方半步外，双手托着卷轴，袖口垂在手腕两侧。窗外冷蓝月色落在后景窗棂上，桌案边缘有暖色烛光。
```

## 八、V6 新增终检项

每组完成后，必须增加以下终检：

1. 纯 Prompt 是否只写当前镜头可见、可听内容？
2. 是否出现“为了表现、用于表现、暗示、说明、代表、意味着、基于参考图、不要、避免、可以、或者、必要时”等污染词？
3. 是否把资产用途说明写进了纯 Prompt？
4. 是否把检查层判断写进了纯 Prompt？
5. 是否写了观众看不见的心理、动机、身份真相、剧情因果？
6. 是否在远景、背影、遮挡镜头中写了不可见表情或手部细节？
7. 是否每个镜头只有一个主动作和一个主空间关系？
8. 是否因为想让模型理解剧情而堆叠了过多解释？
9. 是否存在人物突然出现、场景背景混用、门窗角色混乱、方向词无参照物的问题？
10. 是否符合“所见即所得”第一原则？

只要以上任一项不合格，必须回到检查层重写，不得直接进入纯 Prompt 交付。
