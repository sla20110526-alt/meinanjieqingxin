# 生产闸门 V8.3.1-A Seedance 正式输出前强制执行

规则状态：V8.3.1 的强制执行模块。它不替代《短剧AI视频生产规则_V8.3.1_Codex修正版.md》，只作为所有 Seedance2.0 正式 Prompt 输出前的生产闸门。

适用范围：Seedance2.0 主镜头组 Prompt、覆盖镜头 Prompt、局部返工 Prompt、补拍 Prompt。

执行原则：未过闸门的 Prompt 不得作为正式版本输出。除非用户明确要求“只看草稿”“只检查”或“给我问题清单”，否则输出给用户的必须是已过闸门版本。

## 0. 固定流程

正式输出 Prompt 前必须按以下顺序执行：

1. 草拟镜头结构。
2. 执行生产闸门 1-10。
3. 发现问题时先内部修正。
4. 只输出通过闸门的正式版本。
5. 输出后附一次 V8.3.1 证据式二次规则检查。

不得把未通过自检的 Prompt 先作为正式版本输出，再在用户追问后承认不合格。

## 闸门一：镜头功能闸门

每个镜头先定义唯一核心功能：

- 地点建立
- 入场
- 站定
- 下跪
- 扶起
- 台词
- 反应
- 道具
- 离场
- 空镜
- 转场

一镜只承担一个核心动作或一个核心信息目标。连续动作链超过一个节点，必须拆镜或删减。

错误示例：

```text
走入、走到、站定、下跪。
```

处理：拆成入场、站定 / 下跪、台词等独立镜头。

## 闸门二：上一组尾帧承接闸门

每组开头必须先读取上一组尾帧：

- 人物位置
- 站 / 坐 / 跪状态
- 朝向
- 手部状态
- 道具状态
- 视线方向
- 离场 / 入场方向

本组第一镜必须从上一组尾帧开始。本组尾帧必须明确写出，供下一组承接。

重点检查：

- 并排站立
- 并排跪下
- 扶起后平排站立
- 拉手后分开
- 离场方向
- 道具换手
- 坐姿起身
- 跪姿起身

不得自动跳位、自动站起、自动坐下、自动换手、自动移动到另一侧。

## 闸门三：场景资产方向闸门

每镜先锁定一个正确场景资产方向，再写正文。

常用方向职责：

- 外全景：外观、环境、入场动势、门窗光源。
- 背视：从主位方向看门口、入场路径、凤昭背坐、离场方向。
- 主视：从入口看主位、跪拜关系、权力压迫。
- 侧位：人物台词、扶起、提醒、拉手、手部动作。
- 主位：主位坐姿、起身、屏风、主桌、主位光源。

不得用一个场景方向硬写所有信息。

## 闸门四：景别可见性闸门

当前景别、机位、方向看不见的内容不得写进正文。

外景中远景不得写：

- 室内主位细节
- 屏风细节
- 室内人物脸
- 具体侍从动作
- 室内手部动作

近景不得写：

- 完整空间
- 门口
- 桌案整体
- 远处人物动作
- 大范围群像调度

手部近景只写：

- 手
- 袖口
- 局部光影
- 接触状态
- 局部道具

如果必须表达不可见信息，改到下一镜、覆盖镜头或声音层。

## 闸门五：所见即所得语言闸门

纯 Prompt 只写可见画面和可听声音。

必须删除：

- 剧情解释
- 导演意图
- 策略词
- 抽象判断
- 规则提醒
- 二选一
- 不确定表达
- 不可见信息

控制模型时写可见结果，不写禁令式正文。

正确示例：

```text
凤昭背向镜头坐在主位椅上，只见后背、发饰轮廓、浅蓝衣袖和椅背。
```

错误示例：

```text
不要给凤昭露脸。
不能看清凤昭正脸。
```

## 闸门六：动作负载闸门

每镜最多一个主动作。

人物走位和镜头运动不能同时复杂。每镜最多一个运镜；每组通常只安排 1-2 个轻微运镜。

高风险动作链必须拆：

- 走入 + 走到 + 站定 + 下跪
- 起身 + 走出 + 扶起 + 说话
- 拉手 + 反应 + 道歉 + 推走

## 闸门七：台词与口型闸门

每个可见开口镜头只保留一个说话人。

2.5-4 秒台词镜只保留一句核心台词。

长台词优先拆为：

- 说话人镜头
- 听者反应
- OS
- 下一镜

OS 镜头必须写嘴唇保持闭合。

台词镜头不得叠加复杂走位和复杂运镜。

## 闸门八：覆盖镜头完整性闸门

每条覆盖镜头必须逐条具备：

1. 编号
2. 持续时间
3. 场景资产使用名
4. 景别
5. 机位 / 运镜
6. 可见主体
7. 唯一主动作或环境变化
8. 声音状态

组级音频模式不能替代单条覆盖镜头声音状态。

覆盖镜头不计入主镜头组总时长，但每条覆盖镜头必须有自己的持续时间。

## 闸门九：光影融合闸门

本组必须有光影继承。

镜头正文只写当前可见光影结果。

光源必须来自场景内可见或剧情明确的实景光源。

人物、服装、道具、地面、背景共享同一套光源。

禁止把“人物可读”写成：

- 独立补光
- 均匀亮脸
- HDR 提亮
- 无来源轮廓光
- 美容补光

## 闸门十：输出前二检闸门

正式输出前必须先内部执行：

```text
A. 格式硬检
B. 导演连续性实检
C. 修正版判断
```

格式硬检至少检查：

- 主镜头组总时长是否不超过 15 秒。
- 单镜头时长是否为 1 秒或 0.5 秒精度。
- 每镜第一场景名是否命中资产锚定等号前名称。
- 每镜是否最多一个运镜。
- 生成约束 / Negative Prompt 是否使用 V8.3.1 固定写法。
- 覆盖镜头是否逐条满足完整结构。

导演连续性实检至少检查：

- 上一组尾帧是否被承接。
- 上一镜状态是否能接本镜状态。
- 人物是否自动换位、自动站起、自动坐下或自动换手。
- 动作动词是否符合站 / 坐 / 跪状态。
- 场景资产方向是否支持当前动作。
- 景别是否看得见正文写入内容。
- 台词是否被复杂动作或复杂运镜干扰。

未通过时必须先修正，不得输出未通过版。

## 最小输出纪律

用户要求“给 Prompt”时：

- 输出过闸门后的正式 Prompt。
- 附一次简明 V8.3.1 二次检查。

用户要求“先检查”时：

- 只输出问题清单和修正方向。
- 不输出新 Prompt。

用户要求“只在对话里显示”时：

- 不写 DOCX。
- 仍执行生产闸门。
